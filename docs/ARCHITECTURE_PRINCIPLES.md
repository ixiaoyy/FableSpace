# FableMap 架构原则与边界 v1.0

## 文档目的

这份文档是 FableMap 所有技术决策的**最高优先级过滤器**。

任何新功能、新任务、新协议在进入开发前，都必须先通过本文档的原则检验。

---

## 核心定位

> **FableMap 是现实城市的神话操作系统，不是传统地图 App，也不是全 AI 生成游戏。**

我们的护城河来自：
1. **世界记忆厚度** - 长期玩家-地点关系
2. **可计算镜头系统** - 同一地点的多重解释
3. **真实空间约束** - OSM 地理骨架的不可坍缩性
4. **持续演化能力** - 玩家写回驱动的世界变化

---

## 混合内核原则 (Hybrid Core)

### 必须保持稳定的层

以下层是 FableMap 的**不可坍缩底座**，任何技术趋势都不应动摇：

#### 1. Reality Kernel（现实内核）
- OSM / Overpass 真实地理数据
- 坐标、道路、POI、zone、route 稳定锚点
- 切片边界与世界对象 ID
- **原则**: 优先追求可验证、可缓存、可重建

#### 2. Structured World State（结构化世界状态）
- `world.json` 及其可演化结构
- 写回事件流
- 地点状态与玩���状态
- 最近回声与归档历史
- **原则**: 世界真相必须可持久化、可回放

#### 3. Deterministic Surface Mapping（确定性表面映射）
- `fantasy_type -> asset key`
- `vibe_profile -> lens / pack`
- tile / icon / layer 的可寻址映射
- 前端主舞台的稳定渲染结果
- **原则**: 同样输入必须产生同样视觉结果

### 应交给 AI 增强的层

以下层是 AI 的**主战场**：

#### 4. Semantic Runtime（语义运行时）
- 现实地点如何被翻译成异世界语义
- 一个地点在不同镜头下的意义差异
- **原则**: AI 输出优先结构化字段，不只生成散文

#### 5. World Orchestrator（世界编排器）
- 此时此地应触发何种反馈、事件、广播与暗示
- 哪些内容该被强调、延迟、隐藏或唤醒
- **原则**: AI 负责编排，不负责创造空间

#### 6. World Memory Graph（世界记忆图谱）
- 玩家与地点之间形成了什么关系
- 哪些历史片段应当再次浮现
- 城市如何"记住"某个玩家
- **原则**: 记忆必须可检索、可总结、可触发

#### 7. Generative Surface（生成式表现层）
- scene capsule、分享卡片、地点插图
- 任务微演出、音景与短时氛围层
- **原则**: 这层不承载世界真相，允许局部生成，失败不破坏主世界

## 决策过滤器

### ✅ 应该做的事

| 类型 | 示例 | 原因 |
|------|------|------|
| **编排能力** | 世界编排器、镜头引擎、事件触发系统 | 增强 AI 对世界的理解与响应能力 |
| **记忆能力** | 世界记忆图谱、行为到意义编译器、玩家-地点关系索引 | 形成长期护城河 |
| **解释能力** | 语义运行时、多镜头解释、lens schema | 让同一地点拥有多重可计算意义 |
| **局部生成** | Scene capsules、地点卡片、历史切入演出 | 增强表现力，不破坏稳定性 |
| **写回协议** | P5 写回、P3 治理、P4 时间褶皱 | 让世界可演化 |

### ❌ 不应该做的事

| 类型 | 示例 | 原因 |
|------|------|------|
| **全 AI 生成主世界** | 逐帧生成地图、视频流式世界 | 破坏长时一致性与玩家痕迹附着 |
| **无锚点生成** | 脱离 OSM 的自由生成空间 | 失去真实地理约束的护城河 |
| **替代地理骨架** | 用 AI 生成道路、建筑位置 | 破坏 Reality Kernel 稳定性 |
| **纯表现层堆砌** | 更多图标包、更多视觉效果（在编排层缺失时） | 不形成 AI-native 能力 |
| **传统 App 功能** | 不需要 AI 参与也能做好的功能 | 偏离 AI-First 定位 |

## 技术选型原则

### 1. AI 能力集成

- **优先**: 结构化输出（JSON schema）、长期记忆（embedding + graph）、编排逻辑（rule + LLM）
- **谨慎**: 端到端生成（视频流、完整场景）、实时生成（高成本、低稳定性）
- **禁止**: 让 AI 输出成为世界唯一真相来源

### 2. 数据持久化

- **必须**: 所有世界状态可落库、可回放、可缓存
- **优先**: 结构化数据 > 纯文本 > 二进制 blob
- **禁止**: 只能一次性消费的 prompt 幻觉

### 3. 前端渲染

- **必须**: 确定性渲染（同样输入 → 同样画面）
- **优先**: 2D Canvas/SVG > 3D > 视频流
- **允许**: 局部生成式增强（不影响主舞台稳定性）

## 任务准入检查清单

在认领任何新任务前，必须回答以下问题：

1. **是否符合混合内核原则？**
   - [ ] 不破坏 Reality Kernel 稳定性
   - [ ] 不破坏 Structured World State 可持久化
   - [ ] 不破坏确定性表面映射

2. **是否属于 AI-native 能力？**
   - [ ] 需要 AI 参与才能做好（编排/记忆/解释/局部生成）
   - [ ] 不是传统 App 也能做的功能

3. **是否挂接真实空间锚点？**
   - [ ] 与 `poi / zone / route / slice` 绑定
   - [ ] 不脱离现实地理骨架独立漂浮

4. **是否可持久化与可回放？**
   - [ ] AI 输出可落库
   - [ ] 世界状态可缓存
   - [ ] 玩家痕迹可附着

5. **是否属于四层模型中的正确层？**
   - [ ] Reality Kernel / Structured World State（稳定层）
   - [ ] Semantic Runtime（AI 解释层）
   - [ ] World Orchestrator（AI 编排层）
   - [ ] Generative Surface（AI 表现层）
