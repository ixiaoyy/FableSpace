# FableMap 空间平台主线

本文档定义 FableMap 当前 P0 主线。产品简报见 [PRODUCT_BRIEF.md](PRODUCT_BRIEF.md)，实现分层见 [ARCHITECTURE.md](ARCHITECTURE.md)，数据结构见 [WORLD_SCHEMA.md](WORLD_SCHEMA.md)。

## 主线定位

FableMap 不是传统地图 App，也不是 AI 聊天室列表，而是**真实地图锚点上的 AI 空间平台**。

每个空间都绑定真实地理位置，由店主配置主题、NPC、玩法、访问规则和 LLM。探索者进入空间后，由 NPC 接待、引导、陪伴或主持轻量玩法，并在回访中延续记忆和关系。

## P0 闭环

```text
坐标 / 城市 / 地点
  -> 发现附近镜像空间
  -> 查看空间类型、氛围、NPC 和第一分钟玩法
  -> 进入空间
  -> NPC 主动引导
  -> 对话 / 选择 / 任务 / 创作
  -> 写回访客状态、记忆或状态卡
  -> 回访时延续关系
```

## 核心角色

| 角色 | 产品职责 |
|------|----------|
| 探索者 | 发现空间，进入体验，与 NPC 互动并回访。 |
| 店主 | 创建空间，维护 NPC、玩法、访问规则和 LLM 配置。 |
| NPC | 作为空间接待者、主持者、陪伴者、记录者或玩法推进者。 |

说明：同一个真人可以有店主能力，但游客前台主路径不是“创建控件”，而是“探索空间”。

## 空间是什么

空间是挂在真实坐标上的可进入体验。它至少包含：

- `lat/lon` 真实坐标。
- 店主确认的名称、描述、主题或氛围。
- 访问规则：公开、密码或私密。
- 一个或多个 NPC，格式优先兼容 SillyTavern。
- 可选世界知识、玩法定义、技能包和记忆机制。
- 店主私有 LLM 配置。

空间可以承载陪伴、线索调查、小委托、创作工坊、仪式感体验、展示空间等，但不应变成传统战斗 RPG、通用社交群或导航 POI。

## 探索者体验

探索者进入空间前，应先知道：

- 这个空间大概是什么类型。
- 是否公开、需要密码或不可进入。
- 有哪些 NPC 或主持角色。
- 第一眼应该做什么：打招呼、接任务、找线索、开始仪式、继续回访。

进入空间后，体验重点是：

- NPC 不是被动等用户闲聊，而是主动给出第一步。
- 对话只是互动方式之一，也可以是选择、轻任务、创作输入或回访确认。
- 记忆、状态和关系应服务于下一次回来，而不是一次性消费。

## 店主体验

店主负责空间主权：

- 选择真实坐标。
- 配置空间名称、描述、类型、氛围、公开边界。
- 创建或导入 NPC。
- 配置 LLM 后端、模型和 API Key。
- 发布 / 停用轻量玩法。
- 审核 AI 草稿、状态候选和需要确认的结构化变化。

平台可以降低创作门槛，但不能越过店主最终确认。

## AI 草稿边界

AI 草稿是临时辅助，不是发布内容。

必须满足：

- 店主确认前不进入公开 Tavern payload。
- 店主确认前不覆盖已有 NPC、WorldInfo、玩法或访问规则。
- 店主确认前不随空间包导出。
- 确认后按现有 Schema 保存，不新增隐式字段。

适合做成草稿的内容：

- NPC 角色卡字段。
- 开场白候选。
- 玩法文案候选。
- 状态卡或 GM Layer 候选。

不适合自动草稿发布的内容：

- 店主正史设定。
- 访问控制和 LLM 配置。
- 访客私密信息。
- 平台级规则。

## 玩法边界

允许：

- Tavern-local 的轻量文字玩法。
- 店主确认的玩法定义。
- 访客自己的玩法会话。
- AI Director 或 fallback 推进。

不允许：

- 战斗、等级、装备、排行榜、下注、竞技匹配。
- 引入任意脚本执行或不可控游戏中心。
- 把玩法会话变成公开社交状态。

玩法记录应可落库、可回放、可测试。

## 关系与社交边界

FableMap 关注探索者与空间 / NPC 的关系。

允许：

- 访客私有记忆、回访状态、关系强度、状态卡。
- 店主审批的单空间 NPC 认领或公开关系。
- Owner / system 范围内的空间 / 角色关系边。

不做：

- 好友、私信、动态墙、陌生人匹配。
- 跨空间访客社交图谱。
- 全局在线状态。

## 安全与隐私

- 店主 LLM API Key 只对 owner 可见，不能返回给访客。
- Token 使用量只供店主参考，不做平台结算。
- 私密空间和 Home 信息不能进入公开发现列表。
- 对话、记忆、状态卡和玩法会话要按空间与访客边界隔离。
- AI 不应生成医疗、法律、金融结论或要求现实危险行动。

## 验收标准

一个新增能力只有满足以下条件，才算符合空间平台主线：

- 强化坐标 -> 空间 -> NPC -> 互动 -> 记忆 -> 回访链路。
- 不绕过店主主权。
- 不违反 [WHAT_NOT_TO_BUILD.md](WHAT_NOT_TO_BUILD.md)。
- 有明确 Schema、存储边界和 API 权限。
- 可验证：至少有真实命令、测试、截图或人工验收依据。
