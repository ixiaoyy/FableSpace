# FableMap 产品概述

## 一句话定位

**FableMap 是一个赛博酒馆平台：地图上的每个区域都是一个虚拟场所，用户可以开店、配置 AI NPC、接待访客。**

---

## 为什么做

现有产品往往停留在两个极端：

- 传统地图产品有空间结构，但没有角色连续性、故事张力和记忆沉淀
- AI 对话产品有表达能力，但缺少真实地点锚定、空间上下文和回访连续性

FableMap 的目标是把两者连接起来，同时提供一个新的 UGC 模式：

- **每个人都可以在真实地图上开一间自己的赛博酒馆**
- 酒馆内的 NPC 由 AI 驱动，能和访客自然对话
- 酒馆内容由主人创作不由平台生成，实现真正的用户共创

---

## 它最终想成为什么

FableMap 最终想成为一个**赛博酒馆 UGC 平台**：

- **地图即世界**：真实地图上的每个区域都可以成为一个"酒馆"
- **酒馆即内容**：每个酒馆的内容（角色、场景、对话）由主人创作
- **AI 即灵魂**：酒馆内的 NPC 默认由 AI 驱动，也可以在店主审批下由玩家临时扮演
- **Token 即燃料**：酒馆的运营燃料是 LLM token，由主人自行承担

换句话说，FableMap 不是"AI 生成游戏地图"，而是"每个人都可以在地图上开一间自己的赛博酒馆"。

---

## 核心价值观

| 价值观 | 含义 |
|--------|------|
| **真实锚点** | 酒馆必须坐落在真实地图上，不能是虚假的虚构位置 |
| **主人主权** | 酒馆的内容、角色、氛围、访问规则完全由主人决定 |
| **最小平台** | 平台只做地图、注册、搜索、入口，不管酒馆内部的内容和对话 |
| **开源可移植** | 角色卡格式兼容 SillyTavern，数据可导出，不锁定用户 |
| **真实连接** | 酒馆可以反映真实地点的真实氛围（学校、派出所、社区），也可以完全虚构 |

---

## 与旧方向的核心差异

| 维度 | 旧方向 | 新方向 |
|------|--------|--------|
| **平台角色** | 内容生成方（平台生成幻想世界） | 内容平台方（用户生成酒馆内容） |
| **内容来源** | AI 生成 + OSM 数据 | 人工配置 + AI 驱动 |
| **核心体验** | 探索平台生成的世界 | 在酒馆中和 AI NPC 对话 |
| **商业模型** | 不明确 | Token 消耗由主人承担（简化版） |
| **用户角色** | 探索者 | 探索者 + 店主 + NPC 扮演者 |
| **叙事风格** | 幻想/奇幻 | 现实映射 + 自由创作 |

## 用户角色与核心场景

FableMap 有三类用户角色，同一个用户可以同时扮演多种身份：

```
用户
 ├── 探索者 (Explorer)
 │      打开地图 → 浏览酒馆 → 进入感兴趣的酒馆 → 和 NPC 聊天
 │
 ├── 店主 (Tavernkeeper)
 │      打开地图 → 选择地点 → 创建酒馆 → 添加角色 → 配置 LLM → 开门迎客
 │
 └── NPC 扮演者 (Performer)
        进入开启 hybrid 的酒馆 → 申请认领已有 NPC → 等待店主审批 → 在酒馆内扮演该角色
```

### 场景 A：店主开店

1. 探索地图，找到一个喜欢的地点（或选择任意空白坐标）
2. 点击"创建酒馆"
3. 填写酒馆基本信息：名称、场景描述
4. 设置访问权限：公开 / 密码 / 私人
5. 添加 NPC 角色：导入 SillyTavern 角色卡 JSON，或手动填写
6. 配置 LLM 后端：选择 backend、填入 API Key、选择模型
7. 保存 → 酒馆上线 → 地图上出现标记 → 游客可以进入

### 场景 B：探索者逛街

1. 打开地图，看到散落在各处的酒馆标记
2. 点击感兴趣的酒馆
3. 查看酒馆简介和角色列表
4. 进入酒馆：公开直接进入，密码需要输入
5. 在酒馆内选择 NPC 开始聊天，NPC 默认由酒馆主人的 LLM 驱动；若店主开启 hybrid 模式，部分 NPC 可由审批通过的玩家扮演
6. 离开酒馆，下次还可以再来（NPC 会记住之前的对话）

### 场景 C：店主维护

1. 打开"我的酒馆"面板
2. 查看酒馆状态：访客数量、Token 使用统计、对话记录
3. 管理角色：添加、编辑、删除
4. 管理访问：修改权限、更新密码
5. 配置 LLM：更换 API Key、切换模型、调整参数

---

## 目标用户

### 店主（Tavernkeeper）

- 喜欢创作角色和故事的人
- 对 SillyTavern 角色卡熟悉的用户
- 想在真实地图上创建私人世界的创作者
- 愿意自己承担 LLM token 成本的 AI 爱好者

### 探索者（Explorer）

- 对"现实地点变成可进入故事场景"有强烈兴趣的人
- 喜欢角色互动、地点叙事、都市传闻、轻跑团或酒馆式体验的人
- 喜欢在熟悉街区中发现隐藏故事、对话和情绪入口的人

---

## 为什么它可能有传播力

- **低门槛**：任何人都可以从自己熟悉的真实地点创建酒馆
- **强代入**：地点是自己的，角色却是活的
- **内容可持续**：角色、对话、记忆天然形成连续内容
- **展示性强**：非常适合 demo 视频、酒馆展示和社区传播
- **AI 优势明确**：AI 用于角色扮演、对话生成，而不是硬做地图绘制
- **UGC 模式**：每个人都可以是创作者，不只是消费者

---

## 当前产品策略

当前阶段不再追求"平台自动生成世界"，而是验证下面这条闭环：

> **地图浏览 → 酒馆发现 → 进入酒馆 → 配置 AI NPC → 对话互动 → 关系记忆 → 回访反馈**

只要这个闭环成立，FableMap 就具备继续扩展为 UGC 平台型项目的基础。

在这个基础上，中长期会进一步扩展为：

- 多酒馆连锁经营
- 酒馆搜索与发现
- 公共酒馆排行与推荐
- 跨酒馆事件传播

---

## 当前取舍

当前明确降低优先级的方向：

- 平台自动生成酒馆内容
- 复杂的地图视觉资源包体系
- 生成式场景画面
- "地图是否看起来像 RPG 城镇图"这一类审美目标

当前明确抬升优先级的方向：

- 真实底图接入
- 酒馆创建与配置
- SillyTavern 角色卡导入
- LLM 对话接入
- 访问控制（公开/密码/私人）
- 写回与记忆

---

## 一句话总结

FableMap 现在最应该成为的，不是一个"AI 自动生成地图"的项目，而是一个**每个人都可以在真实地图上开赛博酒馆、与 AI NPC 对话的 UGC 平台**。
