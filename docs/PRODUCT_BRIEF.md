# FableMap 产品简报

## 一句话定位

**FableMap 是世界的镜像面：基于真实地理位置，进入不同类型的 AI 空间，找到属于你的私密空间。**

## 为什么做

传统地图告诉用户“现实里有什么”，但难以提供可回访的角色、故事和陪伴；单一 AI 聊天产品又缺少真实地点带来的入口、氛围和空间归属。FableMap 连接这两者：用真实坐标组织入口，用 AI NPC 让空间可互动、可游玩、可记住用户。

## 当前主体验

```text
当前位置 / 城市 / 地点
  -> 发现附近镜像空间
  -> 选择感兴趣的空间类型
  -> 进入空间
  -> NPC 引导游玩 / 对话 / 任务 / 创作
  -> 记忆沉淀
  -> 回访私密空间
```

## 核心角色

| 角色 | 目标 | 关键动作 |
|------|------|----------|
| 探索者 Explorer | 找到愿意回访的空间 | 发现、进入、互动、留下记忆、回访 |
| 店主 Tavernkeeper | 创作和维护空间体验 | 选择坐标、配置 NPC、玩法、访问规则和 LLM |
| NPC | 主持空间体验 | 接待、引导、陪伴、解谜、发布委托、记录连续性 |

同一个真人可以同时是探索者和店主，但前台主路径不是“游客创建控件”，而是“探索者进入空间体验”。

## 核心原则

- **真实坐标是锚点**：空间必须挂接真实地理位置；地图是入口和氛围来源，不是传统导航产品。
- **空间由店主创作**：主题、NPC、玩法、访问规则和公开边界由店主决定。
- **AI 是 NPC 引擎**：AI 负责对话、主持、记忆和体验推进，不替平台自动发布空间内容。
- **草稿必须确认**：AI 可生成未发布草稿，但店主确认前不得进入公开 payload、不得覆盖已有内容。
- **关系重于社交**：核心是探索者与空间 / NPC 的长期关系，不做通用社交网络。
- **数据可迁移**：角色卡优先兼容 SillyTavern，数据应可导出。
- **Token 自主**：店主自行承担 LLM token；平台不做充值、结算或抽成。

## 当前优先级

优先做：

- 附近 / 地点 / 城市维度的空间发现。
- “这个空间怎么玩”的第一分钟展示。
- NPC 主动引导、陪伴、任务推进和回访记忆。
- 店主后台的空间配置、发布确认和 LLM 配置。
- 轻量文字玩法、状态卡、访客记忆等可回放机制。

降低优先级：

- 复杂纯地图交互、路线规划、传统 POI 搜索。
- 无边界访客社交、私信、动态墙、陌生人匹配。
- 只围绕聊天堆功能而不说明空间玩法。
- 战斗、等级装备、竞技排行榜等传统游戏化系统。

## 典型场景

**探索者游玩**：打开地图，看到附近空间，进入一个感兴趣的地点，由 NPC 说明玩法并开始对话、选择、线索或委托。系统记录可回访的状态，下次回来时延续关系。

**店主供给**：店主选择真实坐标，配置空间主题、NPC、玩法、访问规则和 LLM。AI 草稿可辅助创作，但必须由店主审核后发布。

**NPC 主持**：NPC 不只是等待闲聊，而是承担接待员、GM、陪伴者、创作助手或记录者职责，服务于空间体验和回访连续性。

## 一句话总结

FableMap 要成为真实世界另一面的一组可进入 AI 空间：每个空间由人创作，由 NPC 主持，让探索者找到能被记住、能反复回来的地方。
