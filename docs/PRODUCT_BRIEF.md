# FableMap 产品概述

## 一句话定位

**FableMap 是世界的镜像面：基于真实地理位置，进入不同类型的 AI 空间，找到属于你的私密空间。**

---

## 为什么做

每个人的想法、喜好、情绪和想进入的空间都不同。传统地图产品能告诉用户“现实里有什么”，但很难提供可回访的角色、故事、陪伴和私密体验；单一 AI 对话产品又缺少现实地点带来的氛围、入口和空间归属。

FableMap 要连接这两者：

- 用真实地理位置组织空间入口，而不是做复杂纯地图 App。
- 用 AI NPC 让空间变得可游玩、可陪伴、可探索、可回访。
- 让不同类型的空间满足不同人的喜好：陪伴、探索、任务、创作、仪式、展示等。
- 让探索者最终找到一个愿意反复回来的私密空间。

---

## 它最终想成为什么

FableMap 最终想成为一个**基于地理位置的多类型 AI 空间游玩平台**。

- **世界的镜像面**：现实地点附近可以存在另一层空间入口。
- **地理位置是锚点**：空间基于真实地理位置背景组织，但地图不是主玩法。
- **空间即体验**：每个空间都由店主配置主题、NPC、玩法形式、访问规则和氛围。
- **NPC 即主持者**：NPC 不只是聊天机器人，也可以接待、引导、陪伴、出题、发布委托、记录回访。
- **私密回访**：核心不是无边界社交，而是探索者与空间 / NPC 的长期关系和记忆沉淀。

换句话说，FableMap 不是“AI 聊天室列表”，也不是“传统地图 App”，而是“在现实世界的另一面，进入不同类型 AI 空间游玩”。

---

## 核心价值观

| 价值观 | 含义 |
|--------|------|
| **地理锚点** | 空间基于真实地理位置组织；不要求复杂纯地图交互，但不能脱离现实位置背景。 |
| **探索者游玩优先** | 前台体验优先服务探索者发现、进入、游玩、回访，而不是让游客创建控件。 |
| **空间类型多样** | 空间不只聊天，可承载陪伴、探索、任务、创作、仪式、展示等不同形式。 |
| **店主主权** | 空间内容、NPC、玩法、氛围、访问规则最终由店主确认和决定。 |
| **NPC 主持体验** | NPC 是空间体验的接待者、引导者、陪伴者、主持人和记忆承载者。 |
| **私密关系** | 核心是人与空间 / NPC 的关系，不做无边界访客社交网络。 |
| **开源可移植** | 角色卡格式优先兼容 SillyTavern，数据应可导出，不锁定用户。 |

## 与旧方向的核心差异

| 维度 | 旧口径 | P0 新口径 |
|------|--------|-----------|
| **平台隐喻** | 地图上开空间 / 开店 | 世界的镜像面 / 不同类型的 AI 私密空间 |
| **主体验** | 创建空间、配置 NPC、聊天 | 探索者进入空间，由 NPC 引导游玩、探索、陪伴和回访 |
| **地图作用** | 真实底图与地图浏览 | 地理位置锚点、附近空间入口、氛围来源 |
| **空间形态** | 以聊天为主 | 多类型体验：陪伴、线索、任务、创作、仪式、展示 |
| **用户角色** | 探索者 + 店主 + 额外玩家扮演角色 | 探索者 + 店主 + NPC |
| **社交边界** | 容易扩展到玩家扮演和访客社交 | 聚焦探索者与空间 / NPC 的私密关系 |

## 用户角色与核心场景

FableMap 只有三类核心产品角色：

```text
用户 / 系统
 ├── 探索者 (Explorer)
 │      发现附近空间 → 选择类型 → 进入空间 → 跟随 NPC 游玩 → 留下记忆 → 回访
 │
 ├── 店主 (Tavernkeeper)
 │      创建空间体验 → 配置 NPC / 玩法 / 访问规则 / LLM → 发布给探索者
 │
 └── NPC
        空间内的 AI 角色与体验主持者：接待、引导、陪伴、解谜、委托、记录、记忆承载
```

### 场景 A：探索者游玩

1. 打开 FableMap，看到基于当前位置 / 城市 / 地点组织的镜像空间。
2. 按兴趣选择空间类型：陪伴、探索、任务、创作、仪式、展示等。
3. 进入空间，先看到“这个空间怎么玩”和推荐 NPC。
4. NPC 主动引导第一步：开场问题、线索、选择、委托或仪式。
5. 探索者通过对话、选择或任务推进体验。
6. 系统记录可回访的记忆、关系或状态。
7. 下次回来时，空间和 NPC 能接住上次留下的内容。

### 场景 B：店主供给

1. 店主选择一个真实地理位置背景。
2. 选择空间类型与氛围：陪伴、调查、小委托、创作工坊、仪式空间等。
3. 配置 NPC 角色、开场引导、玩法说明和访问规则。
4. 可使用 AI 草稿辅助，但草稿必须经店主确认后才会发布。
5. 发布后，探索者可以进入空间游玩和回访。

### 场景 C：NPC 主持

1. NPC 根据空间类型承担明确职责，而不是只等待用户闲聊。
2. NPC 可作为接待员说明玩法、作为 GM 推进任务、作为陪伴者记录状态、作为创作助手整理灵感。
3. NPC 的记忆与回应服务于空间体验和回访连续性。

---

## 目标用户

### 探索者（Explorer）

- 想在熟悉或附近地点发现“另一面”的用户。
- 喜欢 AI NPC 互动，但不满足于纯聊天的人。
- 喜欢轻探索、陪伴、线索、任务、仪式感或私密回访体验的人。
- 希望找到一个能记住自己、能反复回来的空间的人。

### 店主（Tavernkeeper）

- 想创造空间体验的人，而不是只创建普通聊天室的人。
- 喜欢角色、故事、氛围和轻玩法设计的人。
- 愿意维护自己的 NPC、访问边界和 LLM 配置的人。
- 熟悉或愿意兼容 SillyTavern 角色卡格式的创作者。

---

## 为什么它可能有传播力

- **发现感**：现实地点附近竟然有另一层空间。
- **多样性**：不同空间对应不同喜好，不只聊天。
- **私密感**：不是公共社交广场，而是能找到自己的角落。
- **回访感**：NPC 记得我，空间能接住上次留下的内容。
- **展示性**：每个空间都可以用“类型 + NPC + 第一分钟玩法”做短视频或分享卡。
- **UGC 供给**：店主创造空间体验，探索者消费和回访。

---

## 当前产品策略

当前阶段验证这条 P0 闭环：

> **当前位置 / 城市 / 地点 → 发现附近镜像空间 → 选择感兴趣类型 → 进入空间 → NPC 引导游玩 → 对话 / 选择 / 任务推进 → 记忆沉淀 → 回访私密空间**

只要这个闭环成立，FableMap 才具备继续扩展空间类型、店主管理和长期回访关系的基础。

---

## 当前取舍

当前降低优先级：

- 复杂纯地图交互、路线规划、传统 POI 搜索。
- 把游客主路径做成创建控件 / 创建空间流程。
- 只围绕聊天堆功能，而不说明空间怎么玩。
- 无边界访客社交、私信、动态墙、陌生人匹配。
- 战斗、等级装备、竞技排行榜等传统游戏化系统。

当前抬升优先级：

- 附近 / 地点 / 城市维度的空间发现。
- 空间类型与“第一分钟怎么玩”的清晰展示。
- NPC 作为主持者的引导、陪伴、任务和记忆能力。
- 探索者回访、私密记忆和长期关系。
- 店主后台的空间体验配置与发布确认。

---

## 一句话总结

FableMap 现在最应该成为的，是一个**基于地理位置的多类型 AI 空间游玩平台**：在世界的镜像面，让探索者进入不同形式的空间，与 NPC 游玩并回访，最终找到属于自己的私密角落。
