# FableSpace 产品简报

## 一句话定位

**FableSpace 是真实世界的故事镜像：选择一个游玩身份，去见一个角色，进入角色所属的完整世界，并在回访中续写彼此的故事。**

## 为什么做

传统地图告诉用户“现实里有什么”，但难以提供可回访的角色、故事和陪伴；单一 AI 聊天产品又缺少真实地点带来的入口、氛围和空间归属。FableSpace 连接这两者：用真实坐标组织入口，用 AI NPC 让空间可互动、可游玩、可记住用户。

## 当前主体验

```text
选择访客私有的游玩身份与自声明性别
  -> 角色优先发现：先选择想见的 NPC
  -> 查看并进入角色所属的完整空间
  -> NPC 结合游玩身份、角色设定和空间场景引导互动
  -> 对话 / 选择 / 任务 / 创作
  -> 记忆沉淀
  -> 回访私密空间
```

角色是主入口，Space 是角色不可拆分的完整世界。每个 NPC 都属于一个挂接真实坐标、拥有稳定规则、当前事件和后果的 Space；真实坐标是世界在现实地图上的锚点与入口。位置、城市和附近入口只作为发现补充与氛围来源。

### 首批系统故事世界

首批公开内容不再以功能型“小铺 / 服务站”充数，而是三座原创、可持续推进的完整故事 Space：

| Space | 类型 | 中央事件 | 访客切入 |
|---|---|---|---|
| 长明宫·雪夜诏书 | 架空古代宫廷 | 皇帝昏迷、寝殿封锁，大太监与公主争夺五更前的开门权 | 乞丐熟悉不在宫册上的水门与暗路 |
| 青槐驿·狐火借命 | 古代东方志怪 | 化形狐狸精想用百年修为救下曾救过自己的书生 | 访客的破碗能盛延缓药效的月光水 |
| 临川大学·最后一堂课 | 现代校园 | 温柔女老师替学生停职，富二代学生试图用钱摆平试卷泄露 | 跨时代访客不受校园身份与金钱规则影响 |

首发先控制为每个世界 2 个简单、对立清楚的角色，优先验证人物是否好聊、世界是否有代入感，再逐步扩展角色。每座世界仍需有稳定规则、当前章节、外部压力、关系与有代价的选择。以上内容均为平台原创系统种子，不依赖照搬书籍、影视作品或既有 IP。

### 首页单角色历史试验

首页当前不把三座原创世界的六位角色一次全部铺开，而先验证一个更明确的入口：玩家选择“乞丐”身份后，只看到安妮，并直接进入她所属的 `history_broad_street_water_1854`。

安妮是**真实历史背景中的原创角色**：约十岁的伦敦穷人家女孩，不是真实历史人物，也不影射任何真实未成年人。她所属 Space 锚定 1854 年伦敦 Soho 宽街霍乱事件。玩家进门时，她不会先讲历史知识，而是抱着空陶罐向玩家讨水；玩家要先决定是否给水、追问水源、找别处的水或求助成年人。John Snow、病例地图、宽街水泵与泵柄决策随后通过证词和选择逐步出现。

这个试验只改变首页的首个推荐入口，不删除三座原创世界，也不改变其 Space / Character ID、既有对话、记忆或默认发现页契约。历史 Space 使用同一套真实坐标、固定正史、角色卡、世界知识、玩法与私有连续性边界，不建立独立的“历史知识产品”或无锚点剧本列表。

主要游玩人群以移动端为主。首页与 Space 对话的产品判断采用“移动优先、桌面扩展”：手机首屏先出现处境、角色请求和主动作，进入后直接看到 NPC 首句与输入区；桌面端可以增加氛围与辅助信息，但不能改变或拖长这条核心链路。

## 核心角色

| 角色 | 目标 | 关键动作 |
|------|------|----------|
| 玩家 / 探索者 | 找到想见、愿意回访的角色与世界 | 选择身份、发现角色、进入、选择、留下记忆、回访 |
| NPC | 以鲜明立场推动所属世界的当前事件 | 提问、交易、拒绝、隐瞒、揭示线索、承担后果 |
| 系统内容 | 提供经过评审、可版本化的完整故事世界 | 维护坐标、正史、角色卡、规则回退与迁移 |

店主 / 创作者相关能力是既有兼容面，不是当前产品角色，也不进入 P0 主链路。当前设计和开发判断一律先服务玩家、角色、故事连续性与回访。

探索者可以在进入主体验前选择一个**游玩身份**。游玩身份是访客私有的虚构扮演上下文，不是新的账号权限、平台角色或公开社交资料。首版只提供 `beggar`，前台只显示“乞丐”；“古代”保留为该身份的内部背景属性，不单独展示。玩家同时需要自声明性别；当前选择界面只开放 `male` / `female`，平台不得从姓名、头像或对话推断。

## 核心原则

- **真实坐标是锚点**：空间必须挂接真实地理位置；地图是入口和氛围来源，不是传统导航产品。
- **角色属于完整世界**：角色不能脱离 Space 单独成为聊天壳；主题、正史、当前事件、规则和后果必须一致。
- **身份服务于扮演**：游玩身份只作为访客私有运行时上下文；它可以改变角色态度和机会，不能改写角色卡、空间正史或访问边界。
- **角色必须有差异**：同一玩家面对两个角色时，应看到不同欲望、秘密、语言、开价与命运，不允许只替换姓名。
- **AI 是 NPC 与连续性引擎**：AI 负责对话、选择推进、记忆和回访，不在运行时擅自创造或覆盖已发布正史。
- **关系重于社交**：核心是探索者与空间 / NPC 的长期关系，不做通用社交网络。
- **数据可迁移**：角色卡优先兼容 SillyTavern，数据应可导出。
- **体验可达且包容**：移动端和窄屏必须可用，关键操作支持键盘，文本保持可读对比度；新增动效应尊重 reduced-motion，界面不得暴露访客或 owner 私密数据。

## 当前优先级

优先做：

- 首页游玩身份选择，以及角色优先的 NPC 发现；角色卡明确展示所属完整空间和现实位置语境。
- 附近 / 地点 / 城市维度的空间发现，作为角色发现和空间氛围的补充入口。
- “这个空间怎么玩”的第一分钟展示。
- NPC 结合玩家游玩身份主动引导、陪伴、任务推进和回访记忆。
- 轻量文字玩法、状态卡、访客记忆等可回放机制。
- 三个首发世界的角色差异、故事推进、私有连续性和回访质量。
- 首页单角色历史试验中，从安妮的具体请求进入 1854 年宽街事件，并验证“先行动、后解锁知识”的吸引力。

降低优先级：

- 复杂纯地图交互、路线规划、传统 POI 搜索。
- 无边界访客社交、私信、动态墙、陌生人匹配。
- 只围绕聊天堆功能而不说明空间玩法。
- 战斗、等级装备、竞技排行榜等传统游戏化系统。
- 店主后台、创作者增长、UGC 供给和平台级内容市场的新功能。

## 典型场景

**探索者游玩**：先选择游玩身份和自声明性别，再从角色中选择想见的 NPC，进入该角色所属的完整空间。NPC 从自身欲望、秘密、场景和当前事件出发感知玩家身份，给出线索、交易、拒绝或选择。系统记录可回访的私有状态，下次回来时从上次结果继续。

**NPC 主持**：NPC 不只是等待闲聊，而是承担接待员、GM、陪伴者、创作助手或记录者职责，服务于空间体验和回访连续性。

## 一句话总结

FableSpace 要成为真实世界另一面的一组可进入故事：玩家因一个角色进门，因一次有后果的选择留下，再因角色记得自己而回来。
