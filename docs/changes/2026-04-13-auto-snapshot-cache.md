## 变更概述

实现地图快照自动缓存，下次回访同一切片时跳过 AMap tiles 请求。

## 行为变更

### 首次进入切片

1. 加载地图 → 显示在线地图
2. 地图瓦片加载完成后（AMap `complete` 事件），自动保存快照到后端
3. 自动切换到本地快照视图

### 回访同一切片

1. 加载快照清单 → 发现已有快照
2. **跳过 AMap 地图创建**，直接显示本地快照
3. 无 AMap tiles 网络请求

### 手动操作

- "抓取当前快照"按钮仍可用，用户可手动刷新快照
- "切回在线地图"按钮仍可用

## 变更影响文件

- `frontend/src/WorldMap.jsx` — 添加 auto-snapshot 逻辑
- `frontend/src/mapAdapter/AMapAdapter.js` — 添加 `_getMap()` 方法用于访问 AMap map 实例

## 技术细节

- 使用 `autoSaveTriggeredRef` 确保每个切片只自动保存一次
- 监听 AMap `complete` 事件，确保瓦片加载完成后再保存
- 5 秒兜底：如果 `complete` 事件未触发，直接保存
- `preferSnapshot` 和 `snapshotManifest` 联合判断：两者都为 true 时跳过地图创建

## 风险

- 快照是静态图片，不支持缩放、拖拽等交互
- 快照视角固定为保存时的视角和缩放级别
- 快照过期：地点切片数据变化后，快照可能与实际不符（用户可手动刷新）

## 验证方式

1. 进入任意切片 → 等待自动保存 → 确认状态显示"本地快照已保存"
2. 刷新页面 → 确认直接显示快照（无需等待 AMap tiles 加载）
3. 切换到不同切片 → 确认地图正常加载并自动保存
