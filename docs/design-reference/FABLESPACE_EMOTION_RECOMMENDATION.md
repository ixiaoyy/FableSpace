# FableSpace 首页改版：情绪推荐架构

> 核心洞察：Steam 卖的是"游戏"，FableSpace 卖的应该是"情绪"

## 设计哲学转变

### Steam 的逻辑

```
用户自述：今天想玩黑神话 / CS / 文明
平台响应：推荐对应游戏
```

### FableSpace 的逻辑

```
用户自述：我累了 / 失恋了 / 想冒险
平台响应：推荐对应情绪空间
```

**不是"今天去哪"，而是"今晚想成为谁"或"今晚想待在哪里"**

---

## 首页模块设计

### 第一屏：情绪入口（核心）

**模块名**：今晚想去哪？

**模块定位**：用户打开 App 的第一个动作

**设计原则**：用户不需要想"去哪"，只需要承认"什么感觉"

```
┌─────────────────────────────────────────────────────┐
│  🌙 今晚想去哪？                                      │
├─────────────────────────────────────────────────────┤
│                                                     │
│  [🙂 放松一下]  [😢 有点孤独]  [🕯 安静待会]          │
│                                                     │
│  [🗝 来点神秘]  [📖 听个故事]  [🎭 角色扮演]          │
│                                                     │
└─────────────────────────────────────────────────────┘
```

**为什么这样设计**：

1. **降低决策门槛**：用户不需要想"哪个空间好玩"，只需要选一个感觉
2. **情感共鸣优先**：承认情绪 → 被理解 → 想进去
3. **和地图产品完全区分**：高德/百度是"去哪"，FableSpace 是"什么感觉"

**用户点击路径**：

```
点击 [😢 有点孤独]
  → 进入：海边长椅 / 深夜天台 / 无人咖啡馆
  → 空间描述："今晚没人，只有风"
  → 触发进入
```

---

### 第二屏：我的常去（个性化）

**模块名**：还记得这里吗？

**模块定位**：给回访用户看的情感锚点

```
┌─────────────────────────────────────────────────────┐
│  你最近来了 3 次                                       │
├─────────────────────────────────────────────────────┤
│                                                     │
│  [🏠 深夜食堂] — 老板记得你第5次来了                   │
│  [🌊 海边长椅] — 上次你说想看日落                      │
│  [🎬 废弃电影院] — 凌晨有新的录像带                    │
│                                                     │
└─────────────────────────────────────────────────────┘
```

**为什么这样设计**：

1. **制造记忆感**：不是"你收藏的空间"，而是"记得你的空间"
2. **叙事化呈现**：每个空间都有自己的"剧情"
3. **促使用户回访**：看到上次的状态，触发"今天想继续"

---

### 第三屏：今晚特别（时间感知）

**模块名**：今晚限定

**模块定位**：利用时间/事件制造稀缺感

```
┌─────────────────────────────────────────────────────┐
│  今晚限定                                              │
├─────────────────────────────────────────────────────┤
│                                                     │
│  [🎃 鬼屋夜] — 只有今晚开放                           │
│  [🎆 烟花祭] — 还有 2 小时                            │
│  [🌸 流星雨] — 正在发生                               │
│                                                     │
└─────────────────────────────────────────────────────┘
```

**为什么这样设计**：

1. **FOMO 驱动**：时间限制创造紧迫感
2. **事件化体验**：不是"逛空间"，是"赶活动"
3. **运营抓手**：店主可以设置限时活动

---

### 第四屏：探索地图（保留入口）

**模块名**：随便逛逛

**模块定位**：给有明确目的地/好奇宝宝看的兜底入口

```
┌─────────────────────────────────────────────────────┐
│  随便逛逛                                              │
├─────────────────────────────────────────────────────┤
│                                                     │
│  [🗺️ 查看地图]                                        │
│                                                     │
│  附近有 12 个空间                                      │
│  ├─ 咖啡馆 3                                          │
│  ├─ 酒吧 2                                            │
│  ├─ 公园 4                                            │
│  └─ 其他 3                                            │
│                                                     │
└─────────────────────────────────────────────────────┘
```

**为什么这样设计**：

1. **保留选项**：不是所有人都需要情绪推荐
2. **降级入口**：当情绪推荐不匹配时，还有地图兜底
3. **信息透明**：让用户知道"附近有什么"

---

### 第五屏：成为店主（商业闭环）

**模块名**：创建一个世界

**模块定位**：转化探索者为创作者

```
┌─────────────────────────────────────────────────────┐
│  创造你自己的空间                                      │
├─────────────────────────────────────────────────────┤
│                                                     │
│  [🪑 创建一个深夜食堂]                                 │
│  [🏠 创建一个秘密基地]                                 │
│  [🌙 创建一个梦中世界]                                 │
│                                                     │
│  已有 1,234 个空间                                     │
│                                                     │
└─────────────────────────────────────────────────────┘
```

**为什么这样设计**：

1. **UGC 正循环**：被体验 → 想创造
2. **降低门槛**：不是"创建空间"，而是"创建一个 XXX"
3. **社交证明**：已有 N 个空间，降低孤独感

---

## 模块顺序的设计逻辑

```
情绪入口 → 我的常去 → 今晚限定 → 探索地图 → 成为店主
    ↑          ↑          ↑          ↑          ↑
  第一眼      回来        今天        兜底       转化
  共鸣       记忆        稀缺        自主        闭环
```

**为什么这个顺序**：

1. **情绪优先**：先解决"不想动脑子"的用户
2. **个人化第二**：给回访用户专属感
3. **时间感知第三**：制造"现在就去"的紧迫感
4. **探索兜底第四**：给"我就是想自己看"的用户
5. **创作者转化第五**：让整个闭环完成

---

## 不做的模块

### ❌ 分类

**原因**：分类是功能型设计，违背情绪优先原则。用户说"我想玩 RPG"而不是"我今天想沉浸在另一个世界"。

### ❌ 热门榜单

**原因**：Steam 的逻辑。情绪体验没有高下之分，"冷门空间"可能正是某人的今晚归宿。

### ❌ 搜索框

**原因**：首页不需要搜索。搜索是兜底功能，应该在二级页面或用户明确需要时出现。

### ❌ 地图全屏

**原因**：地图是工具，不是入口。第一屏给情绪，地图是"随便逛逛"的降级选项。

## 技术实现要点

### 情绪 → 空间 的映射

```python
# 情绪标签系统
EMOTION_TAGS = {
    "放松": ["咖啡馆", "公园", "阳台"],
    "孤独": ["海边", "天台", "深夜食堂"],
    "神秘": ["废弃建筑", "图书馆", "地下"],
    "故事": ["酒馆", "茶楼", "电影院"],
}

# 空间属性
Space {
    id, name, description,
    emotion_tags: List[str],  # 支持多标签
    mood_description: str,    # 氛围描述（给用户看的那句话）
    last_visit_note: str,     # 访客专属记忆
}
```

### 推荐算法（简化版）

```
1. 用户选择情绪 E
2. 查询 tag=E 的空间列表
3. 过滤：用户已去过 → 强化推荐（记得你）
4. 过滤：时间限定 → 置顶（今晚限定）
5. 排序：访客关系强度 + 距离 + 活跃度
6. 返回 Top 3 + 各自 mood_description
```

## 与 Steam 的本质区别

| 维度 | Steam | FableSpace |
|------|-------|----------|
| 推荐维度 | 游戏类型 | 情绪状态 |
| 用户自述 | "想玩 FPS" | "我累了" |
| 平台响应 | 给你游戏 | 给你归属感 |
| 核心价值 | 找到好游戏 | 找到对的空间 |
| 设计隐喻 | 图书馆 | 深夜食堂 |
