## 用户原始洞察

用户提出：把热门电视剧人物放进酒馆里，比如甄嬛传。酒馆里的人物就是剧中的人物，这样可以吸引很多人。

用户举例：甄嬛传是国民级 IP，粉丝众多，很多人想跟"甄嬛"聊天、问"如果我是皇后怎么办"、吐槽剧情。

---

## 核心产品洞察

**用户真正想要的是什么？**

1. **IP 流量**：甄嬛传、琅琊榜、庆余年、金枝欲孽等国民 IP 自带粉丝，不需要从零获取用户。
2. **角色互动感**：粉丝想跟喜欢的角色"说话"，不只是旁观剧情，而是参与对话。
3. **低创作门槛**：店主不需要从零构思角色，IP 角色本身就有完整人设。
4. **二次创作空间**：吐槽剧情、与角色辩论、穿越到某个剧情节点"如果我是皇后怎么办"。

**为什么这在 FableMap 现有产品框架内有价值？**

- FableMap 的核心是"地图 → 酒馆 → NPC 对话 → 记忆回访"。
- IP 酒馆是酒馆的一种形式：店主基于某个 IP 创建酒馆，放入风格相似的 NPC，访客基于真实地图位置来访。
- 访客对话体验完全在现有产品能力范围内，不需要新功能。

---

## ⚠️ 关键产品边界问题：版权

这是这个方向最需要诚实面对的问题。

### 什么不能做

| 做法 | 风险 |
|------|------|
| 平台直接提供"甄嬛"、"雍正"等真实角色名称和设定 | ⚠️ **版权侵权**（郑晓龙/流潋紫版权方） |
| 平台自动生成甄嬛传风格内容 | ⚠️ **版权侵权** |
| 允许任何用户上传甄嬛传角色卡（UGC） | ⚠️ **平台连带责任**，DMCA 风险 |
| 搜索结果中出现"甄嬛传酒馆" | ⚠️ **版权方投诉风险** |

### 什么可以做

| 做法 | 合法性 |
|------|--------|
| 店主**自创**甄嬛风格的原创角色（同名不同设定） | ✅ 店主自担版权风险，平台不介入 |
| 平台提供**通用宫廷 NPC 风格模板**（无具体角色名） | ✅ 无版权问题 |
| 平台提供"宫廷 NPC 性格模板库"（参考甄嬛传但不指名） | ✅ 属于通用类型，非版权内容 |
| 平台与版权方谈**正式授权合作** | ✅ 最干净的方案，但成本高、周期长 |
| 鼓励用户创建**原创 IP 风格**酒馆（避免直接使用版权名称） | ✅ 无风险，且可形成 UGC 生态 |

## 可行方向分析

### 方向 A：通用宫廷 NPC 风格模板（推荐，无版权风险）

**思路**：不直接使用甄嬛传角色，而是：
- 提供"古装宫廷 NPC 性格模板库"（皇帝、嫔妃、太监、宫女、大臣等原型）
- 店主基于这些模板自创角色，给角色起自己的名字
- 酒馆可以是"某朝代风格的酒馆"，访客可以跟"一个甄嬛式的嫔妃 NPC"聊天

**优点**：
- 无版权风险，完全合规
- 模板可复用（宫廷、仙侠、民国、古装均可做通用风格包）
- 店主有创作主权，符合 FableMap 价值观

**缺点**：
- 不是"甄嬛"，是"甄嬛风格的原创角色"，对 IP 粉丝的吸引力打折
- 缺乏 IP 流量加持，需要靠酒馆本身质量吸引用户

**实现路径**：
1. 复用现有 `personalityTemplates.js` 扩展"古装宫廷"分类
2. 首批模板：皇帝（冷酷决策型）、嫔妃 A（温柔腹黑型）、嫔妃 B（直率敢言型）、太监（情报型）、宫女（忠诚守护型）
3. 配套：提供"宫廷酒馆"装修风格建议（不涉及版权内容）

### 方向 B：正式版权授权合作

**思路**：与甄嬛传版权方（郑晓龙/相关版权代理）谈授权合作，平台获得官方授权后可合法提供角色。

**优点**：
- 彻底解决版权问题，IP 价值最大化
- 版权方可能提供官方设定素材

**缺点**：
- 甄嬛传版权归属复杂（流潋紫原著 + 郑晓龙改编 + 乐视/优酷等播放平台权益）
- 授权谈判周期长、成本高，中小团队难以推进
- 其他 IP（琅琊榜、庆余年）需要单独谈

**适用场景**：作为长期目标，FableMap 做大后可考虑。

### 方向 C：用户自创 + 平台不介入（UGC 免责模式）

**思路**：平台明确声明"酒馆内容由店主创作，平台不对内容版权负责"，允许店主上传角色卡。

**优点**：
- 完全不介入版权，风险转移给店主
- 执行成本低

**缺点**：
- 风险并未消失：DMCA 投诉仍可能指向平台
- 平台声誉受损（被版权方标记为侵权平台）
- 搜索和发现功能会让侵权内容更容易被发现

**结论**：⚠️ 不推荐，不解决根本问题。

### 方向 D：粉丝社区 + 原创 IP 借势

**思路**：不做版权角色，而是鼓励粉丝基于某个 IP 的"感觉"创建原创风格酒馆。

- 酒馆名可以叫"本宫酒馆"（不叫"甄嬛传酒馆"）
- 角色可以叫"熹妃"（谐音但不直接同名）
- 重点是**风格和氛围**，不是复制 IP

**优点**：
- 无版权直接冲突
- 粉丝认同感仍然存在（"这明显是甄嬛的感觉"）

**缺点**：
- 法律灰色地带，版权方仍可能主张权益
- 依赖店主自觉，风险参差不齐

**结论**：⚠️ 有风险，需要法律评估。

## 推荐方案：方向 A + 长期方向 B

**短期（立即可做）**：

- 模板不直接使用版权名称，但覆盖宫廷 IP 的核心角色类型

**长期（做大后可探索）**：

- 如果授权太难，可考虑与网文平台合作（如起点中文网），获得 IP 角色的二次创作授权

## 宫廷 NPC 风格模板 MVP 草案

首批 5 个模板，覆盖宫廷 IP 的核心角色类型：

### 1. 冷面决策者（皇帝型）

- 关键词：权力、克制、孤家寡人、少说多看
- 对应甄嬛传：雍正
- 边界：吐槽决策，不攻击身份

### 2. 温柔腹黑型（嫔妃 A）

- 关键词：表面温和、话里有话、不正面冲突
- 对应甄嬛传：皇后/安陵容
- 边界：讽刺话术，不做现实操纵教程

### 3. 直率敢言型（嫔妃 B）

- 关键词：有脾气、敢说、不委屈自己
- 对应甄嬛传：华妃/叶澜依
- 边界：敢言但不人身攻击

### 4. 情报型（太监/宫女）

- 关键词：消息灵通、察言观色、信息就是筹码
- 对应甄嬛传：苏培盛/崔槿汐
- 边界：情报可靠，不造谣

### 5. 忠诚守护型（贴身侍女）

- 关键词：忠诚、有底线、宁死不背叛
- 对应甄嬛传：流朱/浣碧
- 边界：忠诚有底线，不助纣为虐
