# 酒馆发现增强 — 分类浏览与搜索

## Goal

为探索者提供多种发现酒馆的方式，减少对地图散点的依赖。访客可以通过分类标签、关键词搜索、筛选条件快速找到感兴趣风格的酒馆。

## Background & Rationale

当前 `discover.tsx` 只展示地图散点，用户要靠手动放大地图找酒馆，效率低。现有的默认公益酒馆已有标签体系（"公益"、"外星人"、"文游"、"便利店"等），但前端完全没有利用。

产品闭环要求：`地图浏览 → 酒馆发现 → 进入酒馆`。发现层体验差，后面的环节流量会损失。

## Requirements

### 1. 分类标签浏览

在 `discover.tsx` 的侧边栏（或顶部）增加分类筛选区域，基于酒馆/角色的 `tags` 字段（目前公益酒馆已填充）。

**分类维度**（从 tags 提取，首期只做可视分类，不要求精确匹配）：

| 分类 | 对应标签 | 说明 |
|------|---------|------|
| 公益酒馆 | `公益` | 默认公益酒馆，免费体验 |
| 外星便利店 | `外星人`, `便利店` | 《第三货架后面》 |
| 文游委托板 | `文游`, `委托板` | 《午夜委托板》 |
| 古装宫廷 | `古装`, `宫廷` | 风格向酒馆（如果有） |
| 社区陪伴 | `陪伴`, `树洞` | 情感类酒馆 |
| 全部 | — | 不筛选，展示全部 |

**行为**：点击分类标签，侧边栏列表只显示匹配酒馆；多个标签时取并集。

### 2. 酒馆名称搜索

在分类筛选下方增加搜索框，输入酒馆名称实时过滤（前端过滤 `tavern.name`，不需要额外的 API）。

- 搜索是模糊匹配（`includes`），大小写不敏感
- 输入时实时响应，不用提交按钮
- 清空搜索框恢复分类筛选结果

### 3. 公益 vs 普通酒馆筛选

现有 API 支持 `access` 和 `status` 过滤。增加两个切换开关：

- **只看公益酒馆**：筛选 `access=public` + `status=open`
- **只看开放酒馆**：筛选 `status=open`

### 4. API 支持（如需要）

如果分类标签需要服务端过滤（多标签/跨酒馆搜索），后端 `list_taverns` 可能需要新增 `tags` 查询参数。先用前端过滤实现 MVP，API 增强作为后续 scope。

## UI Layout (Draft)

```
[分类标签] [外星便利店] [文游委托板] [公益酒馆] [社区陪伴]
[搜索框: 输入酒馆名称...                    ]
[公益] [开放中]

──────────────────────────────
[酒馆卡片列表 - 同现有 discover 样式]
```

## Out of Scope

- 评分系统
- POI 排序
- 路线导航
- 朋友圈/分享 SDK
- 战斗/游戏化系统
- 跨酒馆社交图谱

## Acceptance Criteria

- [ ] 分类标签点击后，列表只显示对应酒馆
- [ ] 多个标签同时选中时，展示匹配任一标签的酒馆
- [ ] 搜索框输入后实时过滤，不输入显示全部（不受上次搜索影响）
- [ ] "公益酒馆"切换后，只显示公开酒馆
- [ ] "开放中"切换后，只显示 status=open 的酒馆
- [ ] 筛选和搜索可叠加（先选分类，再搜索）
- [ ] 筛选/搜索清空后恢复完整列表
- [ ] 窄屏布局（手机）下筛选区不破坏酒馆列表浏览
- [ ] `npm --prefix .\frontend run typecheck` 和 `build` 通过
- [ ] 不破坏现有地图散点展示
