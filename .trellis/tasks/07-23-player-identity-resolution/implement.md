# 强制登录玩家身份执行计划

## 1. 启动与证据

- [ ] 启动前核对当前工作区 diff，避开玩家状态持久化等并行改动。
- [ ] 记录现有认证回调、会话 Cookie、权限 introspection、根布局门禁和 StoryWorld 匿名 Cookie 的真实调用路径。
- [ ] 确认公开角色发现 / 详情与私有运行时路由的精确清单。

## 2. 后端可信身份边界

- [ ] 将 StoryWorld 公共路由从整段前缀豁免改为精确公开路由。
- [ ] 为私有 StoryWorld 路由复用已验证会话身份，并在调用应用服务前拒绝无效会话。
- [ ] 删除 `fablespace_player_id` 匿名 Cookie 的生成、解析和响应写入。
- [ ] 确保请求头、查询参数、请求体和旧 Cookie 都不能覆盖服务端 `player_id`。
- [ ] 保证所有 `401` 路径在数据库写入前结束，并设置禁止缓存。

## 3. 登录回跳

- [ ] 复用现有 ParallelLines SSO 入口，补充一次性或受签名保护的 `return_to` 传递。
- [ ] 仅允许规范化的本站角色深链；无效值回退 `/`。
- [ ] 回调建立会话后以 `303` 返回安全路径，不泄露票据或会话内容。

## 4. 前端门禁

- [ ] 将根布局整站门禁拆为公开页面可渲染、私有故事动作按需校验。
- [ ] 未登录点击进入故事时显示明确登录动作并携带安全回跳。
- [ ] 私有 API `401` 时刷新会话状态，不自动重放写请求。
- [ ] 登录后重新读取当前轮次，真实呈现恢复或新开始状态。

## 5. 定向验证

- [ ] 验证公开首页与角色详情无需登录。
- [ ] 覆盖每个私有路由的未登录 `401` 与无写入断言。
- [ ] 验证同账号跨设备恢复、不同账号隔离及伪造身份无效。
- [ ] 对 `return_to` 做开放重定向对抗样例。
- [ ] 验证会话过期写请求不落库且不被前端自动重放。
- [ ] 运行 `py -3 -m compileall -q apps/api/src`。
- [ ] 运行 `npm --prefix .\apps\web run typecheck`。
- [ ] 运行 `npm --prefix .\apps\web run build`。
- [ ] 完成桌面与 360px 浏览器登录门禁和回跳验收。

## 6. 收尾

- [ ] `rg` 确认新 StoryWorld 主线不再引用匿名玩家、匿名进度绑定或 `fablespace_player_id`。
- [ ] 核对权威文档、任务 PRD、代码和部署说明一致。
- [ ] 只暂存本任务确认范围内的生产代码与配置，不夹带并行工作区改动。
