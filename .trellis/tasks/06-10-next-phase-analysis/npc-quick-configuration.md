## 背景与问题

### 当前状态
- 店主通过 `addCharacter` API 添加 NPC
- 支持 SillyTavern 角色卡导入
- 缺少快速创建入口

### 问题分析
1. **入口深**: 添加 NPC 入口在管理后台深处
2. **模板少**: 没有预设角色模板
3. **流程长**: 需要填写多个字段
4. **预览弱**: 配置时无法预览效果

### 用户故事
```
作为 店主
我希望 能快速创建一个 NPC
以便 快速填充我的空间内容
```

### API 改动

```typescript
// frontend/app/lib/taverns.ts

// 新增快速创建 API
export async function quickCreateCharacter(
  tavernId: string,
  template: CharacterTemplate,
  overrides: {
    name?: string
    description?: string
    tags?: string[]
  },
  ownerId: string
): Promise<TavernCharacter> {
  const character: Partial<TavernCharacter> = {
    name: overrides.name || template.name,
    personality: template.personality,
    scenario: template.scenario,
    first_mes: template.first_mes,
    tags: overrides.tags || template.tags,
  }
  return addCharacter(tavernId, character, ownerId)
}
```

### 风险

| 模板不够丰富 | 创建效果差 | 持续丰富模板库 |

## 校准补充

### 已核对事实

- 现有 `addCharacter` API 和 SillyTavern 角色卡导入已经覆盖“创建/导入”基础能力。
- 快速配置更适合先做前端模板和表单收敛，不需要新增后端 quick-create API。

### 边界修正

- 模板是平台提供的候选起点，必须由店主确认后才创建 NPC。
- AI 辅助创建只能生成草稿，不得自动覆盖既有 NPC，也不得直接进入公开 Tavern payload。
- 不要从姓名、头像、描述中自动推断或覆盖 `gender`，需保持 `WORLD_SCHEMA.md` 约束。

### 建议 MVP

1. 做“模板选择 -> 最小字段编辑 -> 预览 -> 确认创建”三步 modal。
2. 模板只填 SillyTavern 兼容字段的安全子集：`name`、`description`、`personality`、`scenario`、`first_mes`、`tags`。
3. 创建成功后引导店主进入高级编辑，而不是一次性暴露所有字段。

### 需要确认的问题

- 模板文案由项目内置维护，还是允许店主保存自己的私有模板？
- 首批模板是否按空间类型生成，如 cafe/school/home/convenience-store，而不是泛角色类型？
- 是否要支持批量导入？这可能比快速创建更复杂，建议另起任务。
