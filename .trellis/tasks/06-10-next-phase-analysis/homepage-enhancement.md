## 背景与问题

### 当前状态
- 首页 (`home.tsx`) 只展示 12 个固定空间
- 使用 `buildHomepageView` 生成指标和精选空间
- 缺少动态内容引导和个性化推荐

### 问题分析
1. **入口信息不足**: 用户不知道当前城市有哪些空间
2. **氛围展示薄弱**: 只有封面图和基本信息
3. **缺少行动引导**: 没有明确的下一步操作提示
4. **地理位置感知缺失**: 不显示用户当前位置相关信息

### 用户故事
```
作为 首次访问用户
我希望 在首页快速了解附近有什么空间
以便决定是否深入探索
```

## 风险与依赖

### 风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 地理位置 API 不可用 | 首页位置显示空 | 使用 IP 定位或默认城市 |
| 后端 lat/lon 查询性能 | 查询慢 | 添加缓存/限制返回数量 |

### 依赖

- 后端已支持 `lat/lon/radius` 查询参数（`GET /api/v1/taverns`），实现时需验证排序/距离字段是否满足前端展示
- 地理位置服务（可选，优先用 IP）

## 校准补充

### 已核对事实

- 当前 `frontend/app/routes/home.tsx` 只调用 `listTaverns({ limit: 12 })`，再交给 `buildHomepageView` 生成首页卡片。
- `.trellis/spec/frontend/homepage-dynamic-data-boundary.md` 已要求首页并行获取 tavern cards、platform stats、recent memories；但当前 route 代码还没有完全落实这个契约。
- `GET /api/v1/taverns` 已有 `lat/lon/radius` 参数；不需要先新增 `GET /api/v1/homepage/sections` 才能做附近空间。

### 边界修正

- 首页增强应优先解决“首次用户看懂空间价值”，而不是做推荐系统。个性化推荐、热门标签和类型分布都需要真实行为数据支撑，当前应降级为探索项。
- 定位必须 opt-in；用户未授权时应显示城市/默认坐标或“继续探索公开空间”，不能阻塞首页。
- 不做路线规划、POI 评分、商家入驻等传统地图 App 功能。

### 建议 MVP

1. 先让 `home.tsx` 对齐既有 homepage dynamic data contract：并行请求 `listTaverns`、`getPlatformStats`、`getPlatformRecentMemories`。
2. 在用户主动授权定位后，复用 `listTaverns({ lat, lon, radius, limit })` 展示一组“附近入口”。
3. 搜索入口只负责跳转 `/discover?search=...` 或等价查询，不在首页内实现复杂搜索。

### 需要确认的问题

- 首页是否允许请求浏览器定位？如果允许，默认半径是多少，失败时用哪个城市/坐标兜底？
- 当前首页视觉是否继续沿用 `FableMapHomeReference`，还是允许拆出新的首页组件结构？
- “精选空间”的排序依据是 open/NPC/visit_count，还是需要店主手动推荐位？
