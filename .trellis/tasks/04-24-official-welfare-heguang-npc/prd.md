# 公益酒馆添加和光 NPC

## Goal
把前面提炼出的沟通调停型 NPC「和光」放进现有官方公益酒馆体系中，让访客可以在公益酒馆内找到一个用于关键对话、人际冲突复盘、说服与行动落地的 NPC。

## Requirements
- 将「和光」作为官方公益酒馆内的 TavernCharacter，而不是全站悬浮陪伴助手。
- 优先挂接到现有官方 / 公益 / 默认酒馆数据，不新增脱离真实地图锚点的空间。
- 保持 TavernCharacter / SillyTavern 兼容字段：name、description、personality、scenario、system_prompt、first_mes、mes_example、tags 等。
- 不新增 Schema 字段，不改变现有 API、存储结构、访问控制或 Token 计费模型。
- 内容风格应突出：共同目标、安全感、真诚尊重、关键对话、事实表达、行动落地。
- 不把该 NPC 做成平台自动生成内容能力；这是一次官方预设内容补充。

## Acceptance Criteria
- [x] 现有默认公益酒馆数据中包含「和光」NPC。
- [x] 「和光」字段完整，且不会破坏现有角色数据读取。
- [x] 如项目已有默认酒馆 / 角色回归测试，补充或更新测试覆盖该 NPC。
- [x] 至少运行与改动范围匹配的验证命令。

## Technical Notes
- 预计改动范围：默认酒馆 seed/preset 数据与相关测试。
- 不改前端交互，不新增独立站内陪伴入口。
- 若发现多个公益酒馆候选，优先选择最符合“帮助访客处理现实问题”的官方公益酒馆；若不存在合适酒馆，则在现有默认公益酒馆集合中以最小方式补充一个真实坐标锚定的公益酒馆。
