# 首页角色横滑卡

## Goal

把首页从故事书架改为角色优先的移动端发现入口，并用固定结构的横滑角色卡解决不同角色说明切换时的布局跳变。

## Background

- 用户已确认暖米白、橙色主行动和淡紫交互色的轻小说方向。
- 用户已确认头像只负责定位，切换时整张角色卡横向移动。
- 当前首页已有真实 Space / Character 数据加载和角色图片，组件仍是暗色故事书架。
- P0 首页只展示安妮、魏观海和萧明珠；不使用占位角色补满。

## Requirements

- 只修改首页角色发现组件及其样式，不新增后端、Schema、详情页或聊天能力。
- 使用现有 API 返回的角色姓名、所属故事、简介、标签和头像，不伪造热度、评分、关系数值或续玩记录。
- 头像选择器与横滑卡保持双向同步：点击头像切换对应卡片，手势横滑后选中头像随之更新。
- 所有角色卡使用相同宽高和固定信息槽位；角色文案长度差异不得改变卡片高度或主按钮位置。
- 720px 以下使用横向滑动卡片；720px 至 959px 居中展示卡片并露出相邻项；960px 以上改为左侧角色列表、右侧固定角色卡的主从布局。
- 移动端、平板和桌面端共用同一份角色数据与卡片内容组件，不复制角色映射或路由逻辑。
- 角色主按钮继续使用现有角色入场链接；次级入口继续使用现有故事链接。
- 首页结构随真实角色数量扩展；缺少角色时不渲染占位项。
- 保持移动端优先，桌面使用主从布局，并尊重 `prefers-reduced-motion`。
- 本任务不移除现有全局身份门槛；固定故事身份迁移留给独立任务。

## Acceptance Criteria

- [x] Ready 状态按安妮、魏观海、萧明珠顺序展示真实角色。
- [x] 点击三个头像会把对应角色卡滚动到视口中央，并更新选中态。
- [x] 手势或触控板横滑后，头像选中态与当前居中卡片一致。
- [x] 三张卡的尺寸、文字槽位和 CTA 位置一致，长文本被稳定截断而不引起重排。
- [x] 360px 宽度无页面级横向溢出，关键点击目标不小于 44px。
- [x] 1280px 宽度呈现左侧角色列表与右侧固定角色卡，不把移动端页面直接放大居中。
- [x] 首页不出现故事目录、店主入口、评分、热度、数值好感或占位角色。
- [x] `npm --prefix .\apps\web run typecheck` 与 `npm --prefix .\apps\web run build` 通过。
- [x] 同视口视觉 QA 对照选定稿后无 P0 / P1 / P2 问题。

## Out of Scope

- 新 StoryWorld 深链路由。
- 角色详情、角色聊天、同行模式和关系写回。
- 回忆 / 我的页面和上次同行数据。
- 全局身份选择移除与故事固定身份接入。
