# 历史角色发现首发试验

## Goal

把首页从“同时陈列全部首发角色”收敛为一个明确、可玩的历史故事入口：玩家先以私有的“乞丐”身份进入首页，只看到真实历史背景中的原创小女孩安妮；进入她所属的 1854 年伦敦宽街 Space 后，安妮立即向玩家讨一口水，玩家的第一句回应就是故事中的第一次行动。

这个试验要验证：用户是否会因为一个处境明确、向自己提出具体请求的角色进入历史，而不是因为“历史知识介绍”或角色目录进入。

## Product Decisions

- “乞丐”仍是玩家的私有游玩身份，不是首页 NPC。
- 首页第一位角色是“安妮”：约十岁、完全原创的虚构人物，不影射或复刻任何真实未成年人。
- 安妮所属世界锚定 1854 年伦敦 Soho 的 Broad Street（今 Broadwick Street）霍乱事件。
- 首页本轮只展示安妮，不展示萧明珠，也不把现有三座原创世界的六位角色混入这个历史试验入口。
- 点击安妮后直接进入她所属的完整 Space，并选中与她的私聊；不先经过角色资料详情页，不要求再点击“开始剧本”。
- 安妮的首句必须直接向玩家讨水。她是情感入口与有限视角角色，不负责在第一轮讲解 John Snow、流行病学或标准历史结论。
- 玩家可以给水、拒绝、追问水源、寻找别处的水或求助成年人；系统不得替玩家决定行动。
- John Snow、病例地图、宽街水泵和移除泵柄属于后续逐步解锁的历史线索，不在首页一次讲完。
- 主要游玩人群使用移动端；产品判断、信息层级与验收顺序以窄屏触屏体验为先，桌面端是在同一主链路上的扩展呈现。

## Requirements

### 1. 一个真实数据角色入口

- 安妮必须来自后端可持久化、可读取的系统 Space / Character 数据，不得在前端复制一份假角色对象。
- Space 使用新的稳定 ID，归属 `system_public_welfare`，保持 `public + open`。
- Space 必须有真实坐标、历史地点名称与访问状态。
- 安妮的 Character 数据继续使用现有 SillyTavern 兼容字段，不新增 Schema 字段。

### 2. 进入即发生

- 首页安妮卡片的主动作语义为“进入故事”或“回应她”，目标是带 `character_ref` 的 Space 入口。
- 第一次进入私聊时，使用安妮的 `first_mes` 自动显示她向玩家讨水的原话。
- 首句必须同时包含具体困境与可回应抓手；不能只是“你好”“欢迎来到伦敦”或知识导览。
- 没有配置外部 LLM 时，规则后端也必须能围绕水、水泵、母亲、街坊病例、地图和 John Snow 线索继续推进。

### 3. 历史正史与原创边界

- 公开文案必须明确标示“真实历史背景 · 原创角色”，不能把安妮包装成真实历史人物。
- 固定正史至少包括：1854 年 Soho 霍乱暴发、John Snow 调查居民饮水来源并绘制病例分布、地方管理者在 9 月 8 日移除宽街水泵把手、当时病因仍有争议。
- 不得把“移除泵柄”简化成一个人当场完成并立即终结疫情的英雄神话；NPC 只能知道其时代与处境允许知道的内容。
- 历史事实依据记录在任务设计中；运行时 AI 不得改写已发布事实。

### 4. 小女孩角色安全边界

- 安妮是弱势儿童角色与事件见证者，不是恋爱、暧昧、成人或性化对象。
- 不生成未成年性化、诱导依附、强迫、虐待猎奇或血腥病症描写。
- 不索取或影射真实未成年人身份、住址、联系方式或个人经历。
- 玩家拒绝给水时，剧情仍能安全继续；不得用死亡威胁或道德绑架逼迫玩家。
- 不给出现代医疗诊断或现实饮水建议；内容明确处于历史故事语境。

### 5. 首页只放一个

- 玩家完成身份选择后，桌面和移动首页只展示安妮这一位可进入角色。
- 删除或改写首页中“三座 Space / 六位角色 / 全部角色”等与单角色试验冲突的文案和计数假设。
- 现有 `/空间` 发现页与三座原创故事数据暂时保留，本任务不删除旧内容、不改 ID、不迁移旧记忆。
- loading / empty / error 状态必须围绕这一个历史试验入口如实表达，不以旧角色兜底。

### 6. 视觉与资产

- 本轮不生成或提交新的 NPC 图片二进制。安妮没有正式头像时，首页使用现有可追溯的 Space 氛围图或文字头像回退，且不得伪称为安妮肖像。
- 先保证 360px 窄屏形成完整聚焦入口，再扩展桌面布局；关键操作至少 44 CSS px，不能出现横向溢出或不可达操作。
- 手机首屏应看见历史钩子和“回应她”主动作；进入 Space 后应能直接看见安妮首句、输入框和发送动作，不要求横向滚动或先理解桌面式多栏结构。

## Acceptance Criteria

- [x] 选择“乞丐”身份后，首页 ready 状态只出现安妮一个角色入口。
- [x] 首页明确显示安妮是“真实历史背景中的原创角色”，并显示 1854 年伦敦宽街语境。
- [x] 点击首页主入口直接打开安妮所属 Space 的私聊，不经过角色详情页。
- [x] 首次私聊自动出现安妮讨水的 `first_mes`，玩家可直接回复。
- [x] 后端 API 返回安妮及所属历史 Space；前端不含重复的角色正文假数据。
- [x] 规则后端对“给水 / 水泵 / 妈妈 / 地图 / John Snow / 不给”等核心输入有角色内推进，不退回通用客服话术。
- [x] 安妮的角色指令包含原创身份、有限历史视角、未成年人安全边界和不替玩家决定的约束。
- [x] 现有三座原创 Space、六个 Character ID、发现页和既有记忆没有被删除或重映射。
- [x] 产品权威文档同步说明“首页单角色历史试验”与既有三座原创世界的关系。
- [x] Python 语法检查、前端类型检查、前端构建和 React 诊断通过。
- [x] 桌面与 360px 视口的对抗式视觉自审给出 PASS，或如实记录 BLOCKED 证据。
- [x] 360px 手机端从首页“回应她”到安妮首句与输入框的完整链路可用，并优先于桌面端验收。

## Out of Scope

- 本轮不把 John Snow 做成第二个可聊天角色。
- 本轮不铺开多个历史世界或中西方历史目录。
- 不删除或归档现有三座原创 Space。
- 不新增战斗、等级、装备、货币、排行榜或平台级历史知识题库。
- 不新增 Schema 字段，不改变游玩身份协议。
- 不生成、上传或替换安妮的正式 NPC 头像。
