# 完成回访与关系界面

## Goal

先完成登录后回访、关系摘要、结局摘要、空状态与会话失效的完整响应式界面，只消费现有真实合同。界面完成优先于跨设备数据能力；后者由独立子任务接续。

## Requirements

- 只修改安妮 StoryWorld Character 前台及其现有前端类型/样式，不新增后端接口、字段、迁移或数据库访问。
- 公开角色详情继续允许未登录浏览；开始、恢复、推进和重新开始仍要求有效登录会话。
- 已登录且存在活动轮次时，页面直接恢复真实时间线，并同时展示关系阶段、最近变化原因和已有安全结局摘要。
- 已完成轮次展示本轮结局、已有结局记录和重新开始动作；没有结局记录时只显示简短真实空状态，不制造记忆、日期、次数或统计。
- 首次未登录与写操作期间会话失效必须是不同的可见状态；会话失效后清空未提交输入，不自动重放消息、选择或重新开始请求。
- 加载、未登录、会话失效、读取失败、无活动轮次、活动轮次和已完成轮次使用同一视觉体系。
- 桌面端形成故事时间线与回访侧栏的完整布局；360px 移动端保持时间线、选择和输入优先，所有关键操作可达。
- 只使用已有代码生成视觉与已登记 CDN 资产；本任务不生成或上传图片。
- 不展示内部 affinity、其他玩家信息、旧 Space 合同或解释产品实现的说明文案。

## Acceptance Criteria

- [x] 公开详情、首次登录门禁、会话失效、加载、失败、无轮次、活动轮次和结局状态均有完整且互不混淆的界面。
- [x] 活动轮次从真实 `StoryRun` 展示关系阶段、最近变化原因、时间线、选择与自由回应。
- [x] 有安全结局摘要时展示真实记录；无记录时不渲染伪日期、伪次数、示例记忆或占位结局。
- [x] 会话失效后不显示写入成功，不保留待发送文本，也不自动重放写请求；重新登录动作返回当前角色深链。
- [x] 1280px 有独立回访侧栏，360px 无页面级横向溢出，输入区、选择和重新开始按钮可用。
- [x] 变更不新增 API、Schema、数据库访问或旧 Space 适配器。
- [x] `npm --prefix .\apps\web run typecheck` 与 `npm --prefix .\apps\web run build` 通过。
- [x] 同视口视觉 QA 无 P0 / P1 问题，并记录 PASS / FAIL / BLOCKED。

## Out of Scope

- 长明宫双角色详情与故事交互。
- 新的回访聚合 API、跨设备一致性合同或服务端会话语义。
- 独立“回忆”页面、全局个人中心、公开关系图或社交能力。
- 历史参考资料表面和新增图片资产。
