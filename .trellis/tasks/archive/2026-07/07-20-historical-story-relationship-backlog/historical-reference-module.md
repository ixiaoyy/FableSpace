# 历史资料板块

## 用户价值

有一部分玩家不是只想完成剧情，而是真的想知道：

- 自己正处在哪一年、哪座城市、哪段历史里；
- 中国和世界在同一时间分别发生了什么；
- 剧情中的人物、物件和制度哪些是真实的；
- 哪些角色与对白是为了可玩性创作的；
- AI 总结是否可靠、从哪里可以继续查证。

因此每个历史 Space 可以拥有一个用户主动打开的 `史料` 板块。它是内容表面，不是产品教学，也不取代角色对话。

## 固定信息结构

移动端采用独立页面或全屏抽屉，入口只写 `史料`，不占用聊天首屏。

| 区域 | 必须呈现 | 剧透规则 |
|---|---|---|
| `此刻` | 年月 / 时期、地点、当前历史背景、Space 所在时间线 | 入场前可见 |
| `同时期` | 中国、世界两条时间线；同一时间窗内各 1—3 个重要节点 | 入场前可见；不得暗示无证据的因果 |
| `已发现` | 玩家在对话、物件、证词或选择中接触到的史实条目 | 随游玩解锁 |
| `人物` | 真实人物、原创角色及其身份标记 | 角色出现后解锁；真实人物可提前显示基本身份 |
| `史实与剧情` | 每条内容明确标记 `史实`、`剧情设定` 或 `待核验` | 已发现内容可见 |
| `来源` | 书名 / 馆藏 / 论文 / 权威网页、访问日期、对应条目 | 与史实条目一起可见 |
| `结局史实` | 事件后续与真实结果 | 默认折叠，动作写 `显示结局` |

## AI 与虚构提示

板块顶部固定使用：

> 含 AI 总结；剧情设定可能包含虚构。

该提示是历史内容的来源与完整性状态，不是允许全站重新出现说明文案。

每个内容块还必须有一个状态：

| 状态 | 使用条件 | 可以包含 |
|---|---|---|
| `史实` | 已人工核验，具有可追踪来源 | 日期、地点、制度、人物、物件、原文短引 |
| `剧情设定` | 明确为 FableSpace 创作 | 原创 NPC、未见史料的对话、为玩法设计的具体请求与组合事件 |
| `待核验` | 只有 AI 总结、工作簿概括或来源不足 | 候选说法、存在争议的归因、待确认时间地点 |

硬规则：

- 仅有 AI 总结时不得标为 `史实`。
- 同一段文字不得把史实与虚构混写后只挂一个模糊标签；必须拆成独立条目。
- 真实人物引语没有可核验出处时，不使用引号，不伪装成原话。
- 运行时 AI 不能新建 `史实`；它只能重组已发布资料，并保留条目状态与来源。
- 工作簿中的“历史地位对比”默认是 `待核验` 或编辑问题，不是史实。

## 历史时间线模拟合同

历史 Space 只模拟“人在真实历史里如何选择”，不模拟“历史是否会按另一种方式发生”。

| 边界 | 包含内容 | 玩家与 AI 的权限 |
|---|---|---|
| 固定史实 | 经核验的日期或范围、地点、真实参与者、同场关系、制度阶段、已知公开结果 | 只可经历、发现和理解，不可改写 |
| 可模拟空白 | 原创人物、无史料记载的普通人日常、私人对话、局部选择与关系 | 可以分支，但不得与任何固定史实冲突 |
| 待核验 | 来源不足、存在争议、只有 AI 总结或工作簿概括的内容 | 不作为剧情前提，不由 NPC 说成确定事实 |

创作检查顺序：

1. 先确定固定史实节点和时间精度，再写角色与分支。
2. 检查每个真实人物是否确实可能在该时间、该地点出现；无证据时不安排同场。
3. 检查原创事件是否改变了真实事件的原因、过程、可用技术或公开结果。
4. 检查玩家所有选择的最远后果；任何路径只要能改写固定史实，就不能发布。
5. 把未确定内容退回 `待核验`，不让运行时 AI 根据“听起来合理”自动补齐。

玩家可以拥有“局部但重要”的结果。例如在 1854 年宽街 Space 中，玩家可以给原创角色安妮找水、改变她是否信任自己、决定先问谁并解锁不同证词；玩家不能改变经核验的宽街事件节点，也不能让安妮被描述成真实历史人物。在咸阳变法候选中，玩家可以影响原创小吏与农户的关系；不能决定变法是否发生，更不能让不同时代的比较人物来到现场。

跨时代游玩身份带来的现代知识只能改变 NPC 的怀疑、误解、警惕或私人反应，不能让真实人物提前采用后世技术、制度或结论，并由此改写公开历史。

## 中国 / 世界双线时间轴

### 目标

解决传统历史学习中“中国史、世界史分开讲，难以形成同一时间感”的问题。

### 移动端结构

```text
公元 1069 年

中国
王安石变法 · 开封

世界
西欧城市与商业继续发展

[前一节点] [后一节点]
```

约束：

- 当前 Space 的时间点始终高亮。
- 中国与世界使用同一时间尺度，但移动端纵向堆叠，不强迫横向拖动。
- 每个节点只保留日期、地点、事件名和一条关键事实；展开后再看详细资料与来源。
- “同一时期”只说明时间并列。只有可靠来源证明交流、传播或因果时，才展示连接线。
- 宽时间段使用明确范围，如 `公元 10—14 世纪`，不得伪装成精确同年事件。
- 日期显示必须保留精度：
  - 精确节点：`前221年`；
  - 约数：`约前1300年`；
  - 持续事件：`1519—1522年`；
  - 制度形成：`1729—1732年逐步形成`；
  - 不同阶段：`2016年公投 / 2020年正式退出`。
- 不得为了中西版面对齐，把前594年的梭伦、前508—507年的克里斯提尼和前356年的商鞅合并成“前4世纪同步改革”。
- “诞生、传播、成为官方制度”“发明、改良”“公投、生效”必须拆成不同节点。

## 边玩边学

### 入场前

玩家可以主动打开 `史料`，查看不剧透的：

- 当前年份 / 时期；
- 真实地点；
- 社会与制度背景；
- 同时期中国和世界的重要事件；
- 已确认的真实人物基本身份。

### 游玩中

当玩家真正接触到一项史实，界面只出现一个简短状态动作：

```text
史料 +1
```

它打开对应资料条目，不弹出长篇讲解，不中断角色对话。解锁必须绑定已发布资料 ID 或等价的稳定记录，不能由模型临时生成一段“历史知识”冒充解锁。

### 回访

回访时保留玩家已经发现的资料与来源；没有发现的剧情证据继续隐藏。史料进度属于访客私有学习记录，不形成公开排行榜、等级或知识积分。

## 史实、剧情与角色的示例

以 `CN-A11 南京宝船厂·被划掉的水手名` 为例：

| 内容 | 状态 | 说明 |
|---|---|---|
| 明代曾组织郑和下西洋 | `史实` | 核验后附来源 |
| 宝船厂遗址对应现实坐标 | `史实` | 坐标与馆藏来源核验后发布 |
| 船匠陶娘 | `剧情设定` | FableSpace 原创成年角色 |
| 她弟弟因工伤被划出船册 | `剧情设定` | 中央事件，不声称来自史料 |
| 某类船册的具体划名流程 | `待核验` | 找到制度或档案来源前不能标史实 |

反例：

```text
错误：前3世纪 · 秦统一 / 奥古斯都建立罗马帝国
正确：前221年 · 秦统一
      前27年 · 奥古斯都开启元首制
      编辑关系：帝国转制主题镜像，相隔约194年
```

## 内容生产流程

```text
工作簿 / 选题
  -> 待核验条目
  -> 来源搜集
  -> 史实与剧情拆分
  -> 人工核验
  -> 剧透级别
  -> 中国 / 世界时间轴定位
  -> 发布资料
  -> 对话与选择引用
```

## 未来实现边界

- 本文件不定义新 Schema。
- 不把史料正文塞进 `Space.description`、`scene_prompt` 或角色 `first_mes`。
- 正式实现前需要单独设计：资料条目、来源、状态、时间范围、剧透级别和访客解锁记录的持久化契约。
- 如果 MVP 暂时复用现有 `WorldInfoEntry`，也必须先证明它能稳定区分来源、状态与剧透，不得靠正文约定偷偷扩展字段语义。
- 历史资料属于经过评审、可版本化的系统内容；运行时 AI 只能在已发布资料范围内总结。

## 验收

- [ ] 入口只写 `史料`，不占用移动端聊天首屏。
- [ ] 顶部出现且只出现一次 `含 AI 总结；剧情设定可能包含虚构。`
- [ ] 每条内容有 `史实`、`剧情设定` 或 `待核验`。
- [ ] `史实`条目均有来源；无来源的 AI 文本不能冒充史实。
- [ ] 所有剧情分支均未改变经核验的时间、地点、真实参与者、制度阶段或公开结果。
- [ ] 原创情节位于可模拟空白内，且已检查不会与其他固定史实发生隐性冲突。
- [ ] 玩家提供的跨时代知识没有被转化为改写历史的技术、制度或结果。
- [ ] 当前 Space 时间点在中国 / 世界双线中明确高亮。
- [ ] 同期并列不被误画成因果关系。
- [ ] 时间标签保留精确、约数、范围或阶段形成语义。
- [ ] 主题镜像没有被伪装成同年或同世纪事件。
- [ ] 结局史实默认折叠并标明剧透动作。
- [ ] 解锁资料不打断聊天，也不生成等级、积分或排行榜。
