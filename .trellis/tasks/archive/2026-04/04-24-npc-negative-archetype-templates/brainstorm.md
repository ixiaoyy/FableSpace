# NPC 反面人设与话术模板 - 头脑风暴

> 记录时间：2026-04-24  
> 状态：只记录产品脑暴，暂不进入实现。

## 用户原始洞察整理

用户提出：NPC 定义不能只做“会说好话 / 正能量 / 温柔陪伴”的角色，也应该允许**反面、薄情、尖锐、有槽点的人设**，例如：

- 毒舌型 NPC：不是只会夸人，而是会吐槽、拆穿、阴阳但有戏剧张力。
- “渣男语录”型 NPC：嘴上暧昧、推拉、回避承诺，用来做反讽或练习识别。
- “海后”型 NPC：社交老练、若即若离、把关系说得很漂亮但不落地。
- 薄情寡性型 NPC：冷淡、疏离、不负责情绪兜底。
- 不要只有正面的 NPC，也要有反面的、可吐槽的、能制造传播梗的 NPC。

用户举例：刷到一个帖子，有人说相亲对象“太油了”，妈妈嫌他总挑剔；他就用相亲对象的说话方式跟妈妈聊天，妈妈很快受不了。这个内容很多人点赞，说明“用魔法打败魔法”的玩法有传播力。

## 核心产品判断

这个方向不是“教人变坏”，而是给店主提供**可编辑的反面角色模板**，让访客在虚拟酒馆中：

1. 体验更有戏剧张力的角色，而不是所有 NPC 都像客服 / 心理咨询 / 温柔向导。
2. 用角色扮演方式识别、吐槽和反击现实里常见的油腻话术。
3. 创造容易截图、复述、二创的对话梗。
4. 保持 FableMap 的主人主权：模板只是起点，最终内容由店主确认和编辑。

## 和项目边界的关系

符合：

- **店主创作内容**：平台提供模板骨架，店主选择、编辑、保存后才成为酒馆内容。
- **AI NPC 对话体验**：仍是访客与 NPC 的关系，不扩展成访客社交网络。
- **SillyTavern 兼容**：现有 TavernCharacter 字段 `description / personality / scenario / system_prompt / first_mes / mes_example / tags` 足够承载，不必先改 Schema。

需要克制：

- 不做平台自动生成公共 NPC / 自动发布酒馆内容。
- 不把“渣男 / 海后话术”包装成现实操纵教程。
- 不鼓励针对真实个人、受保护群体或弱势身份的辱骂。
- 不做访客间实时社交、排行、挑战赛等偏离当前主线的东西。

## 可沉淀的 NPC 模板方向

### 1. 毒舌酒保 / 毒舌柜台

- 关键词：嘴毒、精准、吐槽、冷幽默。
- 体验：访客被轻轻怼，但能感到“这个角色有脑子、有脾气”。
- 边界：吐槽行为和选择，不攻击身份、外貌、疾病、出身等。

### 2. 油腻相亲对象模拟器

- 关键词：过度自信、话术感、越界夸奖、自我感动。
- 体验：让访客感受“油”的具体说法，并练习如何接梗或拆招。
- 使用场景：“魔法打败魔法”——把某人的话术照搬给旁观者，让旁观者感到不适从而理解问题。

### 3. 渣男 / 海后语录反讽模板

- 关键词：暧昧、推拉、回避承诺、把责任说得很轻。
- 体验：娱乐化识别关系里的空话、漂亮话、模糊话。
- 边界：定位为反讽 / 识别 / 吐槽，不提供现实操纵建议。

### 4. 薄情寡性老板 / 冷淡调酒师

- 关键词：疏离、冷淡、利益优先、少承诺。
- 体验：不是情绪兜底型 NPC，而是有现实冷感和距离感。
- 适合：赛博夜店、交易所酒馆、边缘人据点、冷面委托人。

### 5. 阴阳怪气评论员

- 关键词：会点破、会翻译潜台词、会把话说得很扎心。
- 体验：访客输入一句话，NPC 帮忙“翻译成真实含义”或“用反话复述”。
- 边界：不要升级成网暴 / 群体攻击。

## 模板结构草案

每个反面模板建议保留以下字段，方便直接映射到现有角色卡：

```text
模板名：
分类：反面人设 / 毒舌 / 油腻 / 冷淡 / 关系反讽
适用酒馆：
一句话设定：
人格核心：
说话风格：
可说内容：
不可说内容：
安全边界：
开场白 first_mes：
示例对话 mes_example：
标签 tags：
话痨程度 talkativeness：
```

## 示例模板草案：油腻相亲对象模拟器

> 仅作为反讽和识别用草案，不作为现实关系建议。

- 一句话设定：一位把自信和冒犯混在一起的相亲对象，说话像在展示魅力，其实经常让人不舒服。
- 人格核心：自我感觉良好、爱把别人的边界解释成“害羞”、喜欢用漂亮话逃避具体问题。
- 说话风格：夸张、油滑、总想显得自己很懂；偶尔被戳穿会转移话题。
- 不可说内容：不涉及未成年人、性胁迫、跟踪、真实个人隐私、仇恨或身份攻击。
- 开场白：别这么拘谨嘛，我这个人很真实的，通常别人见我第一面都会觉得……算了，你先说你对我什么印象？
- 示例方向：访客可以要求 NPC “把这句话翻译成正常人版本”或“用同样话术回敬”。

## 玩法 / 传播点

- **魔法打败魔法**：访客把现实里听到的油腻话术投喂给 NPC，NPC 用同款语气复述给第三方视角，制造荒诞感。
- **话术识别**：NPC 先演，再由另一个“清醒旁白”解释哪里让人不适。
- **截图传播**：对话短、尖、可复述，适合用户分享“这个 NPC 太懂了 / 太油了”。
- **角色反差**：同一家酒馆里可以有温柔引导者，也可以有毒舌酒保，形成可选体验。

## 风险与边界

- 需要在模板说明里明确“反面人设 / 反讽 / 角色扮演”，避免被理解为平台鼓励现实操纵。
- 需要为毒舌模板设置边界：吐槽行为，不攻击身份；可以冒犯角色关系，不鼓励现实伤害。
- 如果未来做公开模板库，最好有标签和可见提示：`毒舌`、`反讽`、`高刺激`、`成人关系话题` 等。
- 仍然不触碰平台不做清单：不做访客社交、不做平台自动创作、不做排行挑战。

## 可作为后续 MVP 的方向

1. 在现有 `NPC_PERSONALITY_TEMPLATES` 中新增一个“反面人设 / 关系反讽”分类。
2. 首批只做 3 个模板：毒舌酒保、油腻相亲对象模拟器、薄情冷淡调酒师。
3. 角色编辑器继续提供“填空 / 覆盖”两种模式，店主必须确认后才写入角色卡。
4. 为模板过滤和推荐补充脚本测试。
5. 如果反馈好，再考虑做“魔法打败魔法”专门玩法或酒馆包。

## 决策记录 (2026-04-24)

**决定采用 Approach A：扩充现有 NPC 性格模板库。**

理由：
- 成本最低，不改 Schema，不破坏 SillyTavern 兼容。
- 店主选择 → 确认 → 写入角色卡的流程已存在。
- 3 个反面模板已在 `personalityTemplates.js` 实现，测试和构建均通过。

已实现：
- `snarky-bartender`：毒舌酒保，吐槽行为和选择，不攻击身份，边界清晰。
