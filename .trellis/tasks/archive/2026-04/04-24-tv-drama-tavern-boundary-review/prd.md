# 热门电视剧人物酒馆 - PRD

> 状态：已完成产品边界评审，当前暂不开发。

## Goal

评估“热门电视剧人物酒馆”是否适合进入 FableMap 当前开发主线，并把结论沉淀到 Trellis，避免后续误把平台做成官方影视 IP 内容生成/分发系统。

## What I Already Know

- FableMap 当前定位是赛博酒馆 UGC 平台：酒馆内容、角色、氛围、访问规则由店主决定。
- `docs/WHAT_NOT_TO_BUILD.md` 明确不做平台自动生成 NPC、酒馆描述、对话内容或故事。
- `docs/FABLEMAP_TAVERN_PLATFORM.md` 明确平台角色是内容平台方，不是内容生成方。
- `docs/WORLD_SCHEMA.md` 的核心角色模型仍是 `TavernCharacter`，并保持 SillyTavern 兼容。

## Product Boundary

“热门电视剧人物酒馆”有传播力，但如果由平台预置或生成具体剧中人物，就会同时触碰两个边界：

- **主人主权边界**：平台不能替店主创作酒馆内容或 NPC。
- **IP/版权边界**：热门电视剧人物、台词、立绘、剧情桥段通常不是项目可自由分发的素材。

因此，当前不实现“平台内置热门剧人物酒馆”，不新增具体影视 IP 角色卡，不生成剧中角色台词或头像。

## Accepted Future Direction

未来可以考虑的安全方向：

- 店主自己导入拥有权利或已获授权的角色卡，平台只做托管与格式兼容。
- 提供原创泛风格创作辅助，例如“宫廷权谋酒馆”“都市群像茶馆”，但不使用具体剧名、具体人物、原台词或可识别设定。
- 在导入/发布流程中加入内容权利提醒，避免把平台包装成官方 IP 内容库。

## Out of Scope

- 平台预置热门电视剧角色。
- 平台生成具体剧中人物角色卡、头像、台词、剧情或世界知识。
- 官方示例酒馆使用具体影视 IP。
- 新增 schema / API / 前端模板库。
- 对版权问题做自动判定或法律审核。

## Decision (ADR-lite)

**Context**：用户提出用热门电视剧人物增强酒馆传播力。

**Decision**：本任务只做产品边界留档，当前暂不开发。

**Consequences**：

- 保持 FableMap “店主创作、平台托管”的定位。
- 避免把平台变成影视 IP 内容生成/分发方。
- 后续若用户明确要做，只能从“授权内容导入”或“原创泛风格模板”重新立项。
