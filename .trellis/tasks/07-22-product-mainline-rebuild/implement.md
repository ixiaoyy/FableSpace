## 7. 验证命令

按子任务范围运行，最终集成至少执行：

```powershell
py -3 -m compileall -q apps/api/src
npm --prefix .\apps\web run typecheck
npm --prefix .\apps\web run build
```

另需：

- 定向 API 脚本验证新故事状态与权限边界；
- 360px 移动端和桌面端真实浏览器验收；
- 对两个 P0 故事的开场、关键选择、关系变化、结局和回访做真实数据验证；
- `rg` 检查旧路由、`Space`、地图、owner、SillyTavern 和旧字段残留；
- 高风险删除前后核对完整 `git diff` 和目标文件清单。

## 8. 回滚点

- 领域与 API 完成但前端未切换：回滚新路由注册即可，不删除旧表。
- 前端切换完成但旧代码未删除：可回退前端入口，新运行时表保持不动。
- 旧代码与表清退：只通过明确迁移回滚或备份恢复，不依赖应用启动自动重建旧数据。
- 任一阶段验证失败时停止进入下一子任务，不用兼容适配器掩盖失败。
