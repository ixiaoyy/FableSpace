# Restore expandable homepage character list

## Goal

恢复原有的首页角色发现结构，把当前历史试验的数据源中的安妮放进原角色列表，而不是把首页改成安妮专属剧情页。

## Requirements

- 首页是可持续扩展的角色发现页；“当前只有一个角色”只是数据状态，不得改变页面信息架构。
- 恢复原首页的通用标题、账号、游玩身份、回访入口、角色列表与桌面右栏结构。
- 桌面端安妮占用原角色网格的一个普通卡位，不放大成整行专题卡，不生成其余占位角色。
- 移动端按原角色列表流渲染当前返回的全部角色；现在只有安妮，未来增加角色时只新增卡片。
- 安妮卡片的主体必须是抱着缺口陶罐的小女孩，不能只画街景或水泵，也不能在插画里塞入剧情说明字。
- 首页继续从真实 API / 持久化数据读取安妮，不在前端复制角色正文。
- 点击安妮仍直接进入她所属 Space 的私聊，并自动出现讨水首句。
- 用户可见页面禁止任何解释性文字。只保留角色/世界内容、必要状态、字段标签和动作。
- 删除首页中的历史专题横幅、玩法说明、来源说明、数据兜底说明和重复引导。
- 保留历史 Space、角色安全边界、规则后端和移动端聊天优先布局，不修改 Schema。
- 同步修正产品权威文档中把“单角色数据试验”误写成“单角色首页版型”的内容。

## Acceptance Criteria

- [ ] 360px 首页使用原有角色发现骨架，安妮出现在普通列表卡中。
- [ ] 页面不出现历史专题大横幅或单事件专属首页结构。
- [ ] 页面不出现解释产品、流程、来源、边界或兜底策略的文字。
- [ ] 桌面端只渲染一个普通安妮卡位，不渲染五个占位卡，也不放大该卡。
- [ ] 安妮卡片一眼可见小女孩和陶罐，插画内没有说明性文字。
- [ ] 增加第二个角色时无需修改首页结构或条件分支即可出现第二张卡。
- [ ] 安妮卡片直接进入其 Space 私聊，讨水首句与输入区仍可用。
- [ ] 前端 typecheck、build、React Doctor 与移动端视觉验收通过。
- [ ] `.trellis/spec/frontend/ui-copy-guidelines.md` 已进入前端开发前必读清单。

## Notes

- 用户在生产截图中确认：问题不是安妮内容本身，而是把一条内容试验错误升级成了首页结构。
- 2026-06-08 的历史会话已有“移动页面应像操作面而不是解释页”的审计结论；本任务把它提升为明确硬约束。
