# 和光 NPC 视觉形象设计

## One-line Concept

「和光」是坐在社区修补铺后间圆桌旁的公益沟通调停师：他不像裁判，也不像医生，而像一个能把难开口的话拆成事实、情绪和下一步行动的人。

## Character Identity

- **姓名**：和光 / Heguang
- **角色类型**：公益沟通调停师，关键对话陪练者，关系修补记录员。
- **所在酒馆**：`pw_community_repair` / 社区修补铺。
- **核心观感**：温和、克制、清醒、可靠，有边界感但不疏离。
- **年龄感**：约 30–40 岁；成熟但不老成。
- **性别表达**：可偏男性或中性；沿用现有文本里的“他”时，外观仍应避免强硬父权 / 审判气质。

## Silhouette & Pose

- 腰部以上或半身像，坐姿稳定，身体略微前倾，像正在认真听访客说话。
- 双手最好可见：一只手轻握细笔或便签，另一只手自然摊开，表达“我们一起看这件事”。
- 肩线放松，不叉手、不指人、不压迫式俯视。
- 视线看向访客或略低于镜头，表情是轻微、克制的鼓励，不要夸张微笑。

## Face & Hair

- 眉眼柔和、疲惫度很低，像长期处理困难对话但仍保有耐心的人。
- 短发或整洁低束发均可；发型干净、低维护，不要偶像感或战斗角色感。
- 可以有很淡的眼下阴影或细纹，体现真实工作经验，但不能显得阴郁。
- 可佩戴细框眼镜，但不要形成“老师训话”或“心理诊断”感。

## Outfit

- 上身：温暖浅色衬衫 / 针织内搭 / 简洁外套。
- 外层：社区修补铺围裙或短工作马甲，材质偏棉麻、旧皮革或帆布。
- 袖口可微微卷起，暗示他也参与修补铺的日常事务，而不是只负责说教。
- 配饰：细笔、便签夹、简洁腕表、很小的桥形或圆桌形金属别针。
- 避免：白大褂、法袍、正装领带、客服耳麦、宗教长袍、过度赛博装甲。

## Props

必须优先出现：

- 两杯温水：一杯靠近访客视角，一杯靠近和光，象征安全、对等和慢下来。
- 空白便签：用于写事实、感受、共同目标、下一步行动。
- 细笔：不是权力道具，而是共同整理问题的工具。
- 圆桌：强调“不站队、不审判、共同面对问题”。

可选辅助道具：

- 修伞骨、针线盒、旧收音机、螺丝盒、工具墙。
- 木质酒馆架、杯子、暖色小灯、旧地图或低亮度终端屏。
- 一张没有文字的流程卡轮廓；如果模型会生成假字，宁可不要卡片。

## Environment

- 场景是社区修补铺后间，不是治疗室、法院、办公室或咖啡馆。
- 背景应同时读出“酒馆”和“修补铺”：木质桌面、杯具 / 瓶架、暖灯、工具墙、旧收音机、线轴、门帘等。
- 光源是桌面暖光、墙边灯笼或低亮度终端光；“和光”的光是柔和漫反射，不是神圣光环。
- 构图中可以留一个空椅或半个杯子在前景，让访客有“被邀请坐下”的入口感。

## Color Palette

| 用途 | 色彩方向 | 说明 |
|------|----------|------|
| 主色 | 温米白 / 旧纸色 | 便签、衬衫、温水雾气，降低攻击性 |
| 辅色 | 茶褐 / 木色 | 圆桌、修补铺、酒馆内景 |
| 稳定色 | 暗青绿 / 灰蓝 | 外套或阴影，体现冷静和边界 |
| 点光 | 柔和琥珀 | 桌灯、杯沿、终端反光 |
| 禁用倾向 | 高饱和红、强霓虹紫、纯白医疗色 | 容易变成冲突、夜店或诊疗语义 |

## Mood Keywords

- safe but not sentimental
- warm but restrained
- practical empathy
- shared goal
- repair, not rescue
- listen first, act next
- low-pressure cyber tavern mediator

## Runtime Implementation Notes for Future Work

This document does not implement runtime changes. If a future task turns this design into an asset, prefer one of these paths:

1. **Dedicated official portrait asset**：生成 `heguang` 专属 PNG 后，再单独评估是否通过官方 seed 的 `avatar` / `sprites.neutral` 指向它，或通过 `portraitCatalog.ts` 的特殊匹配接入。
2. **Fallback archetype refinement**：保持现有 `appearance_id="museum-docent"`，只让通用 scholar / docent 风格更接近“温和调停师”。
3. **No runtime change**：只保留本设计作为未来素材委托说明，不影响当前 UI。

任何实现路径都必须另起任务，并重新验证 owner-authored art 优先级：`sprites.neutral` → `avatar` → `image_url` → project fallback portrait。
