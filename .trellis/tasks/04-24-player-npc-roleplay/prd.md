# 玩家扮演 NPC 角色扮演模式 - PRD

> 状态：头脑风暴中，暂不开发

## 背景与动机

当前 FableMap 是一个 **AI NPC 体验平台**：
- 店主配置 AI NPC
- 访客进入酒馆与 AI 对话
- 核心价值：真实地点 + AI 角色 + 记忆系统

**问题：** 纯 AI 对话的局限性
- AI 缺乏真实的社交感
- 访客参与度有限（只是聊天）
- 无法形成长期的玩家社区关系

**新方向：** 混合社交角色扮演

> 玩家不仅可以"体验"酒馆，也可以"成为"酒馆的一部分。

## 核心玩法

### 角色扮演模式

玩家可以**扮演**酒馆中的 NPC 角色，与其他访客进行角色扮演互动。

```
┌─────────────────────────────────────────────────────────────┐
│                        酒馆场景                              │
│                                                             │
│   ┌─────────┐     ┌─────────┐     ┌─────────┐              │
│   │ AI NPC  │     │ 玩家    │     │ 玩家    │              │
│   │ 酒保    │     │ 扮演    │     │ 扮演    │              │
│   │ (AI)   │     │ 吟游诗人 │     │ 老板    │              │
│   └─────────┘     └─────────┘     └─────────┘              │
│                                                             │
│        ▲              ▲              ▲                       │
│        │              │              │                       │
│   店主配置        玩家抢占        玩家常驻                    │
│                                                             │
│   ┌─────────────────────────────────────────────┐           │
│   │              访客（游客模式）                  │           │
│   │   进来看到：AI NPC + 玩家扮演的 NPC          │           │
│   │   可以和任何人/AI 进行角色扮演互动            │           │
│   └─────────────────────────────────────────────┘           │
└─────────────────────────────────────────────────────────────┘
```

### 角色类型

| 类型 | 来源 | 控制者 | 在线状态 |
|------|------|--------|----------|
| AI NPC | 店主创建 | 店主/AI | 24/7 |
| 玩家扮演 NPC | 抢占或新建 | 玩家 | 按玩家在线 |
| 混合代理 | AI + 玩家混合 | 玩家指导 | 自动切换 |

### 抢占机制

玩家可以"认领"一个角色槽位：
1. **申请扮演** - 向酒馆申请扮演某个角色
2. **审批** - 店主审批（或自动批准）
3. **上线扮演** - 玩家接管该角色的对话
4. **下线代理** - 玩家离线后 AI 接管或标记"暂时离开"

### 常驻身份

玩家可以申请成为某个酒馆的"常驻 NPC"：
- 长期扮演某个角色
- 获得特殊标识（badge/称号）
- 可能需要满足在线时长要求
- 获得访客的认知和关系积累

## 数据模型变更（概念）

### 新增/变更实体

```typescript
// 酒馆模式
enum TavernMode {
  experience,  // 体验模式（当前）
  roleplay,    // 角色扮演模式
  hybrid,      // 混合模式
}

// 角色状态
enum CharacterStatus {
  ai_controlled,     // AI 控制
  player_claimed,    // 玩家认领
  player_online,     // 玩家在线扮演
  ai_fallback,       // AI 代理（玩家离线）
}

// 角色认领记录
interface CharacterClaim {
  character_id: string
  player_id: string
  status: CharacterStatus
  claimed_at: Date
  offline_at?: Date
}

// 玩家角色卡（不同于 SillyTavern）
interface PlayerCharacter {
  player_id: string
  tavern_id?: string  // 可选，绑定到特定酒馆
  name: string
  backstory: string
  personality: string
  tasks?: CharacterTask[]
  reputation: number
}
```

## 技术影响

### 高影响

- **实时通信** - WebSocket/SSE 用于玩家在线状态和即时消息
- **消息路由** - 区分 AI 消息和玩家消息
- **状态同步** - 玩家上线/下线状态同步

### 中等影响

- **数据模型** - 新增 CharacterClaim、PlayerCharacter 等表
- **前端组件** - 角色状态指示器、扮演模式 UI
- **AI 代理** - 玩家离线时 AI 接管逻辑

### 低影响

- **地图/发现** - 可能需要显示酒馆模式
- **搜索** - 搜索角色扮演酒馆

## 风险与挑战

| 风险 | 描述 | 缓解方案 |
|------|------|----------|
| 审核 | 玩家内容不可控 | 店主审核、敏感词过滤、举报机制 |
| 冷场 | 玩家不在线时角色缺失 | AI 代理机制 |
| 冲突 | 多个玩家抢同一角色 | 排队机制、申请审批 |
| 质量 | 玩家扮演质量参差不齐 | 评价系统、激励高质量扮演 |
| 规模 | 需要足够活跃玩家 | 早期聚焦核心社区 |
