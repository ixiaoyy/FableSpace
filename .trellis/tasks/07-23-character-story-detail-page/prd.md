# 铺通安妮故事垂直链路

## Goal

只围绕安妮铺通一条真实、可刷新、可继续的玩家链路，让玩家从首页选择安妮后，依次看到她所属的 1854 年宽街故事、固定“乞丐”身份和讨水开场，并完成一个最小但真实的故事闭环。暂不横向扩展魏观海、萧明珠或通用多故事能力。

## Background

- 首页已经展示安妮、魏观海和萧明珠，但当前主动作仍直接跳入旧 `/stories/:spaceRef?...#story` 页面。
- 旧 `space.tsx` 混合通用 Space 和聊天工作台，并依赖已取消的全局 `VisitorPlayIdentity` 门槛。
- 新 `StoryWorldRegistry` 只有不可变类型和校验器，尚无真实发布内容、公开详情 API 或玩家运行时 API。
- 安妮是原创未成年人角色，故事必须遵守 `.trellis/spec/guides/historical-content-integrity.md`：固定史实、剧情设定、待核验分层，不能让玩家或 AI 改写 1854 年宽街事件的固定结果。
- 用户已确认采用真实数据垂直切片，不接受只做视觉页面壳、前端硬编码或继续扩大旧 Space 适配。

## Requirements

### R1. 单角色范围

- 本轮只新增安妮的真实新链路。
- 魏观海、萧明珠、青槐驿和临川大学不进入本轮新详情、运行时或持久化实现。
- 不为了“以后通用”预先增加无当前验收用途的抽象、字段或页面。

### R2. 真实发布内容

- 将安妮所属的 1854 年宽街内容表示为通过 `StoryWorldRegistry` 校验的已发布 StoryWorld。
- 固定 PlayerRole 为“乞丐”；Character 为原创约十岁女孩安妮。
- 入场从安妮抱着陶罐讨水开始，玩家先面对具体行动，再逐步接触水源、病例和泵柄相关历史线索。
- 固定史实、剧情设定和来源必须通过历史内容对抗式核验；待核验内容不得进入发布因果。

### R3. 首页到详情

- 首页点击“去见安妮”先进入安妮独立详情页，不再直接滚到旧聊天工作台。
- 详情页使用可分享、可刷新恢复的 Character 深链。
- 页面展示安妮、所属 StoryWorld、一句当前处境、固定“乞丐”身份、入场理由和明确进入动作。
- 未登录且没有旧全局身份记录的访客可以查看并进入；不得要求用户选择身份、性别或自定义 Prompt。

### R4. 新合同边界

- 页面只消费新的 StoryWorld / Character 公开只读响应，不新增旧 `Space`、`space_id`、`VisitorState` 或 `/spaces` 兼容适配。
- 公开响应不包含私有进度、对话、关系、记忆、隐藏正史、系统 Prompt 或玩家标识。
- 直接打开无效或未发布引用时返回受控未找到状态，不泄露内部记录。

### R5. 最小故事闭环

- 进入后锁定安妮 StoryWorld 的内容版本和固定 PlayerRole。
- 安妮的首句、玩家可观察动作、关键选择和状态变化来自审核过的故事骨架。
- 自由输入不能绕过关键选择、改写正史或直接制造结局。
- 关系变化只展示态度、阶段和原因，不显示内部 affinity。
- 完成后可以识别该轮结局；重新开始时不继承上一轮好感或故事标记。
- 玩家刷新或回访时，系统恢复属于自己的当前轮次或已完成摘要，不暴露其他玩家数据。

## Acceptance Criteria

- [x] AC1：首页点击安妮进入真实 Character 详情页；魏观海和萧明珠没有被顺手迁移或重写。
- [x] AC2：详情页显示安妮、1854 年宽街、固定“乞丐”身份、故事处境和进入动作。
- [x] AC3：详情 URL 可直接刷新和分享；无效或未发布引用返回受控状态。
- [x] AC4：未登录且没有旧 `VisitorPlayIdentity` 的访客能从首页进入安妮开场。
- [x] AC5：安妮的发布内容通过 StoryWorldRegistry 校验和历史内容对抗式审查。
- [x] AC6：至少一条玩家路径从讨水开场走到已审核结局，关键选择不可回退，固定史实不被改变。
- [x] AC7：刷新或回访可恢复当前轮次或安全结局摘要；重新开始不继承上一轮可变状态。
- [x] AC8：公开接口和页面不依赖旧 Space 合同，不暴露任何其他玩家私有数据。
- [x] AC9：360px 移动端与桌面端的详情、开场、输入/选择和结局动作可用。
- [x] AC10：相关 Python 语法、定向数据库/API 验证、前端 typecheck 与 build 通过。

## Out of Scope

- 魏观海、萧明珠或其他 Character 的新详情和故事运行时。
- 多 StoryWorld 通用目录、全局身份、账号绑定、跨设备合并和社交能力。
- 旧 Space API、表和页面的最终清退。
- 多结局全量内容、并行时间线、章节回退、地图、用户创作或架空历史分支。

## Confirmed Scope

本轮采用最小真实闭环：从首页到安妮详情、固定身份和讨水开场，允许自由回应，并可沿一条人工关键选择路径到达已审核结局；刷新能恢复当前轮次或结局摘要，结局后可以重新开始。
